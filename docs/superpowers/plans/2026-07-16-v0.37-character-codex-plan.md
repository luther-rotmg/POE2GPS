# v0.37 "Book of the Exile" Character Codex — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Bead-eligible tasks dispatch via `bd create --label worker --acceptance ...`.

**Goal:** Ship v0.37 Character Codex: a per-character, local-only, loopback-gated event journal (level-ups, boss kills, deaths, notable drops) with a dashboard "book" UI, built additively on validated boss attribution and a stability-gated persistence sink without expanding game-memory reach or /state's public surface.

**Architecture:** Four-layer additive stack. (1) DISCOVERY layer: D1 codifies live-playtest boss metadata as an xUnit theory exercising BossEncounterCatalog.Shared.ByMetadata so authored matchMetadata strings are proven against reality before any codex code trusts them; D2 lands a per-character rolling event sink (SessionEventLog) modeled byte-for-byte on DropTimeline's persistence seams (load-on-construct, append-on-record, ring-capped, flush-on-dispose) with a character-name STABILITY GATE (N consecutive non-empty identical ObservePlayerName ticks) that defeats the login/zone-load flicker profile of Poe2Live.PlayerName. (2) SCHEMA + EMITTERS layer: C1 defines a System.Text.Json-polymorphic CodexEvent hierarchy (discriminator "kind" ∈ {level,boss,death,drop}); C2 adds a CodexEmit event to SessionTracker firing at the existing level-up gate and death-edge sites; C3 lands a new CodexBossObserver at RadarApp.cs:1690-1692 as a parallel observer to _session.ObserveKill, requiring BOTH Rarity.Unique AND a catalog hit (metadata-first, zone-fallback) with an alive→dead edge detector mirroring KillTracker's _sawAlive semantics; C4 adds a Recorded event to DropTimeline (raised OUTSIDE the _gate lock, only on non-deduped accepts) plus a CodexDropForwarder that emits on Unique-rarity OR first-seen-ItemArt-per-character. (3) API layer: A1 adds a loopback-gated GET /api/codex?character=<name> handler to ApiServer via an injected Func<string,object?> codexProvider, with the loopback host check FIRST (before method check) so LAN peers cannot use HTTP status as an existence oracle; strict regex ^[A-Za-z0-9_-]{1,40}$ kills path traversal at parse; the /state anonymous object stays byte-for-byte unchanged (regression-gated by grep + a leak test). (4) UI layer: A2 adds a Codex tab to DashboardHtml.cs (sentinel-wrapped markup + IIFE JS) that fetches /api/codex, groups events by local-timezone yyyy-MM-dd into "spread" blocks, offers four toggle chips (level/boss/death/drop) held in a page-lifetime closure, and a jump-to-date <select> that scrollIntoView-s the matching spread; ASCII-only labels to defeat the known worker CP1252→UTF-8 mojibake failure mode.

**Tech Stack:** .NET 8 / C# (Core + Overlay projects), System.Text.Json (polymorphic serialization via [JsonPolymorphic] + [JsonDerivedType]), System.Net HttpListener-based ApiServer, xUnit test suite (per-facet Facts + Theory with MemberData), vanilla ES5-safe browser JS + native HTML controls (no framework) inside a C# raw-string DashboardHtml blob, filesystem-backed per-character JSON persistence under {ConfigDir}/codex/, dispatched via the beads pipeline (opencode workers, size:small/medium/complex classes) with dotnet build + dotnet test as the mandatory validate gate on every bead.

## 🚨 Locked Architectural Decisions (READ BEFORE ANY BEAD DISPATCH)

The parallel workflow decomposers produced drift between D2 (persistence layer) and C1 (event type) — same class of issue as v0.36 K2's flat-vs-nested schema. Frontier locks the following BEFORE any bead dispatches so downstream workers can't disagree:

**1. File format: JSONL (one JSON object per line, append-only writes)**

Not a whole-file `{ events: [...] }` envelope. Rationale: append-only writes avoid the read-parse-mutate-serialize cycle DropTimeline uses for its ring; codex event volume is much higher than drops (level-ups, deaths, boss kills, notable drops all pile up over months). Each line is one CodexEvent JSON with a `kind` discriminator.

**2. Event type: polymorphic `CodexEvent` with `kind` discriminator**

```csharp
public abstract record CodexEvent(long Ts, string Kind, uint AreaHash, string Zone);
public sealed record LevelUpEvent(long Ts, uint AreaHash, string Zone, int Level) : CodexEvent(Ts, "level", AreaHash, Zone);
public sealed record BossKillEvent(long Ts, uint AreaHash, string Zone, string BossKey, string BossLabel) : CodexEvent(Ts, "boss", AreaHash, Zone);
public sealed record DeathEvent(long Ts, uint AreaHash, string Zone, int AreaLevel, int PlayerLevel) : CodexEvent(Ts, "death", AreaHash, Zone);
public sealed record NotableDropEvent(long Ts, uint AreaHash, string Zone, string Name, string Rarity, string? Art) : CodexEvent(Ts, "drop", AreaHash, Zone);
```

Serialized via `System.Text.Json` with a `[JsonPolymorphic]` type discriminator on `Kind` OR a small custom `JsonConverter` — C1 picks whichever compiles cleanly on the current .NET target.

**3. File path convention: `{ConfigDir}/codex/<safeName>.jsonl`**

- `ConfigDir` (not `AppContext.BaseDirectory` — matches BossWipeLog pattern; the DropTimeline field-initializer style doesn't work because ConfigDir isn't resolved until ctor body).
- `<safeName>`: `PlayerName` sanitized by replacing filesystem-illegal chars with `_`. Empty name → skip (don't open a file).
- Extension: `.jsonl` (matches format decision above).
- Directory created lazily on first write (mirror DropTimeline.Flush's `Directory.CreateDirectory` pattern).

**4. Character-name stability gate**

D2 must NOT open a file until `PlayerName` has been non-empty AND stable (same string) for `N=30` consecutive session-tracker ticks (~1 second at 30Hz). Prevents opening a codex file for the WRONG character during login flicker.

**5. Boss-kill attribution: strict allowlist (no debug bucket in v0.37)**

Unknown uniques are silently dropped (no `{kind:"boss",label:"unknown"}` events written). Debug-bucket for unknown-uniques deferred to v0.37.x point release if attribution proves too narrow in real play. This is the safest posture for the codex's permanent write path.

**6. D1 is a Frontier-input-blocked task**

D1's xUnit theory rows are LO-provided metadata paths from a real playtest session (~10 known bosses + ~30 non-boss uniques). Frontier CANNOT gather these; they require live PoE2 play. Do not dispatch D1 until LO has provided the answer key.

using System;
using System.IO;
using System.Linq;
using System.Text;
using POE2Radar.Core.Session;
using Xunit;

namespace POE2Radar.Tests.Session;

public class SessionEventLogTests : IDisposable
{
    private readonly string _root;

    public SessionEventLogTests()
    {
        _root = Path.Combine(Path.GetTempPath(), "poe2radar-tests", Guid.NewGuid().ToString("N"));
        Directory.CreateDirectory(_root);
    }

    public void Dispose()
    {
        try { Directory.Delete(_root, recursive: true); } catch { }
    }

    private static SessionEvent Ev(long ts, string kind = "kill", string zone = "MapUberBoss_Arbiter", int lvl = 82, string? payload = null)
        => new() { TimestampSec = ts, Kind = kind, Zone = zone, AreaLevel = lvl, Payload = payload };

    [Fact]
    public void Record_dropped_before_any_stable_name()
    {
        var log = new SessionEventLog(_root);
        Assert.False(log.Record(Ev(1)));
        Assert.Null(log.OpenCharacter);
        Assert.Empty(log.Snapshot());
    }

    [Fact]
    public void Opens_file_only_after_N_stable_ticks_of_same_nonempty_name()
    {
        var log = new SessionEventLog(_root);
        for (int i = 0; i < SessionEventLog.NameStabilityTicks - 1; i++) log.ObservePlayerName("Alice");
        Assert.Null(log.OpenCharacter);
        Assert.False(log.Record(Ev(1)));

        log.ObservePlayerName("Alice");
        Assert.Equal("Alice", log.OpenCharacter);
        Assert.True(log.Record(Ev(2)));
        Assert.Single(log.Snapshot());
    }

    [Fact]
    public void Flicker_to_empty_resets_stability_counter_but_does_not_close_open_file()
    {
        var log = new SessionEventLog(_root);
        for (int i = 0; i < SessionEventLog.NameStabilityTicks; i++) log.ObservePlayerName("Alice");
        Assert.Equal("Alice", log.OpenCharacter);
        log.Record(Ev(1));

        log.ObservePlayerName("");
        log.ObservePlayerName(null);
        Assert.Equal("Alice", log.OpenCharacter); // stays open across blank flicker

        log.ObservePlayerName("Alice"); // only 1 stable tick now — no re-open, still Alice
        Assert.Equal("Alice", log.OpenCharacter);
        Assert.Single(log.Snapshot());
    }

    [Fact]
    public void Character_swap_switches_file_after_new_name_is_stable()
    {
        var log = new SessionEventLog(_root);
        for (int i = 0; i < SessionEventLog.NameStabilityTicks; i++) log.ObservePlayerName("Alice");
        log.Record(Ev(1, kind: "levelup"));
        log.Flush();

        // One-tick flicker of Bob does NOT switch.

```csharp
using System;
using System.Collections.Generic;
using System.IO;
using System.Linq;
using System.Text.Json;
using System.Text.Json.Serialization;

namespace POE2Radar.Core.Session;

public sealed record SessionEvent
{
    [JsonPropertyName("ts")]        public long   TimestampSec { get; init; }
    [JsonPropertyName("kind")]      public string Kind         { get; init; } = "";
    [JsonPropertyName("zone")]      public string Zone         { get; init; } = "";
    [JsonPropertyName("areaLevel")] public int    AreaLevel    { get; init; }
    [JsonPropertyName("payload")]   public string? Payload     { get; init; }
}

/// <summary>
/// Per-character rolling log of session events (deaths, level-ups, boss kills, drops)
/// backing the v0.37 Character Codex. Mirrors DropTimeline's storage seams verbatim:
/// load-on-construct + append-on-record + flush-on-dispose; LinkedList ring cap with
/// oldest-first eviction; corrupt-file → fresh in-memory state; whole-file writes.
///
/// Adds a character-name STABILITY GATE: the log will not open/create a per-character
/// file until the observed PlayerName has been non-empty and unchanged for
/// NameStabilityTicks consecutive ObservePlayerName() calls. This defeats the login
/// and character-swap flicker documented in Poe2Live.PlayerName's cache profile
/// (empty-frame under zone load, address change on swap).
///
/// Not thread-safe against concurrent Dispose; all other members are lock-safe under
/// the internal _gate.
/// </summary>
public sealed class SessionEventLog : IDisposable
{
    public const int MaxEntriesPerCharacter = 5000;
    public const int NameStabilityTicks     = 5;

    private static readonly JsonSerializerOptions Json = new()
    {
        WriteIndented        = true,
        PropertyNamingPolicy = JsonNamingPolicy.CamelCase,
    };

    private readonly string _configDir;
    private readonly object _gate = new();

    // Currently-open character (null = no file open, all Record calls dropped).
    private string?                 _openName;
    private readonly LinkedList<SessionEvent> _entries = new();
    private bool                    _dirty;
    private int                     _generation;

    // Stability-gate state.
    private string? _candidate;
    private int     _stableTicks;

    public SessionEventLog(string configDir) { _configDir = configDir; }

    public int     Generation     { get { lock (_gate) return _generation; } }
    public string? OpenCharacter  { get { lock (_gate) return _openName; } }

    /// <summary>Feed the current-tick PlayerName reading. Null or empty resets the stability counter but does NOT close an already-open file.</summary>
    public void ObservePlayerName(string? name)
    {
        lock (_gate)
        {
            if (string.IsNullOrEmpty(name))
            {
                _candidate   = null;
                _stableTicks = 0;
                return;
            }
            if (name == _candidate) _stableTicks++;
            else { _candidate = name; _stableTicks = 1; }

            if (_stableTicks >= NameStabilityTicks && name != _openName)
            {
                FlushLocked();
                _openName = name;
                LoadLocked(name);
                _generation++;
            }
        }
    }

    /// <summary>Append an event under the currently-open character. Returns false if no character is open yet.</summary>
    public bool Record(SessionEvent ev)
    {
        if (ev is null) return false;
        lock (_gate)
        {
            if (_openName is null) return false;
            _entries.AddLast(ev);
            while (_entries.Count > MaxEntriesPerCharacter) _entries.RemoveFirst();
            _generation++;
            _dirty = true;
            return true;
        }
    }

    public IReadOnlyList<SessionEvent> Snapshot()
    {
        lock (_gate) return _entries.ToArray();
    }

    public void Flush()
    {
        lock (_gate) FlushLocked();
    }

    private void FlushLocked()
    {
        if (!_dirty || _openName is null) return;
        var snap = _entries.ToArray();
        _dirty = false;
        var path = PathFor(_openName);
        try
        {
            var dir = Path.GetDirectoryName(path);
            if (!string.IsNullOrEmpty(dir)) Directory.CreateDirectory(dir);
            var bytes = JsonSerializer.SerializeToUtf8Bytes(new { events = snap }, Json);
            File.WriteAllBytes(path, bytes);
        }
        catch { /* nice-to-have; silent — matches DropTimeline.Flush */ }
    }

    private void LoadLocked(string name)
    {
        _entries.Clear();
        _dirty = false;
        var path = PathFor(name);
        if (!File.Exists(path)) return;
        try
        {
            using var doc = JsonDocument.Parse(File.ReadAllBytes(path));
            if (!doc.RootElement.TryGetProperty("events", out var arr) || arr.ValueKind != JsonValueKind.Array) return;
            foreach (var el in arr.EnumerateArray())
            {
                // Per-element try/catch — schema-drift defense for v0.37 codex events
                // (an improvement over DropTimeline.Load's all-or-nothing behavior).
                try
                {
                    var ev = JsonSerializer.Deserialize<SessionEvent>(el.GetRawText(), Json);
                    if (ev != null)
                    {
                        _entries.AddLast(ev);
                        while (_entries.Count > MaxEntriesPerCharacter) _entries.RemoveFirst();
                    }
                }
                catch { /* skip malformed row, keep valid neighbors */ }
            }
        }
        catch { _entries.Clear(); /* corrupt file → fresh start; never throws */ }
    }

    private string PathFor(string name)
    {
        var safe = new string(name.Where(c => char.IsLetterOrDigit(c) || c is '_' or '-').ToArray());
        if (string.IsNullOrEmpty(safe)) safe = "_unknown";
        return Path.Combine(_configDir, "codex", safe + ".json");
    }

    public void Dispose() => Flush();
}
```

DESIGN DECISIONS worth understanding before editing:
(a) Ring cap eviction runs BOTH on Record AND inside the per-entry Load loop — same defensive pattern as DropTimeline.
(b) `Generation` is bumped on Record AND on open (both are observable state changes for the dashboard poller).
(c) `PathFor` strips every non-alnum/`_`/`-` character; there is no known-real character-name grammar guarantee, so we sanitize hard.
(d) `FlushLocked` runs the whole write inside the lock (unlike DropTimeline which drops the lock before serialize). This bead ships the simpler shape; a follow-up bead can widen the seam if the write hot path shows up in profiling. Documented tradeoff, not an oversight.

    public CodexBossObserver(
        Func<string, BossEncounterEntry?> byMetadata,
        Func<string, BossEncounterEntry?> byZoneCode,
        Action<CodexEvent> sink)
    {
        _byMetadata = byMetadata ?? throw new ArgumentNullException(nameof(byMetadata));
        _byZoneCode = byZoneCode ?? throw new ArgumentNullException(nameof(byZoneCode));
        _sink       = sink       ?? throw new ArgumentNullException(nameof(sink));

    public void ObserveEntityTick(Poe2Live.EntityDot e, string currentAreaCode, long tsUnixSeconds)
    {

        // under-discriminating (see catalog audit finding #4) — we MUST require a catalog hit before
        // emitting.

        var id    = e.Id;

        lock (_gate)

                return;

            if (!_sawAlive.TryGetValue(id, out var wasAlive) || !wasAlive) return;
            _sawAlive.Remove(id); // consume the edge → guards double-emit

        var entry = _byMetadata(e.Metadata ?? string.Empty)
                 ?? _byZoneCode(currentAreaCode ?? string.Empty);
        if (entry == null) return;

        {
            Key   = entry.Key,
            Label = entry.Label,
            Zone  = currentAreaCode ?? string.Empty,
            Ts    = tsUnixSeconds
        });
    }

## Known Risks & Mitigations

1. SHAPE MISMATCH between D2 (SessionEvent record + whole-file .json envelope) and C1 (polymorphic CodexEvent + intended per-line .jsonl). Must file a reconciliation bead — either promote CodexEvent as D2's storage type and switch to JSONL append, or have A1 consume D2's envelope shape. Do NOT dispatch A1 until this is resolved; otherwise the endpoint reads files that don't exist or reads the wrong schema.

2. PATH EXTENSION MISMATCH: D2 writes {ConfigDir}/codex/{safeName}.json, A1 reads {ConfigDir}/codex/{name}.jsonl. Either file will not exist or A1 will read empty. Same reconciliation bead as above.

3. D1 has a hard Frontier-side input dependency — the playtest MUST be run and both metadata arrays captured BEFORE the bead body is written. If dispatched with empty arrays, the worker fails the bead by design (Playtest_input_is_populated Fact). Budget a real play session before D1 goes out.

4. C3 and C4 both edit RadarApp.cs ctor wire-up block. Dispatch serially (C3 first, then C4) or accept a manual merge in the ctor. Parallel dispatch of both is a known nested-closure-arg-style ctor bead failure vector — MEMORY.md flags two prior failures in this exact shape.

5. Emoji / CP1252 mojibake in DashboardHtml.cs is a documented worker failure mode (feedback_worker_emoji_mojibake.md). A2's spec enforces ASCII-only labels ([Lv]/[Bo]/[Dx]/[Dr]) and a grep gate on ≡ƒ/ΓÜ/┬·/ΓÇö plus U+1F300-U+1FAFF and U+2600-U+27BF ranges. Do not accept an A2 diff that fails the grep.

6. A1 modifies ApiServer.cs ctor signature (adds optional codexProvider param) — nested-closure-arg ctor edits are the known failure vector. Spec anchors exact lines and mandates trailing-comma preservation; enforce diff review against the anchor lines before commit.

7. Boss-kill attribution surface predicts 4 of 5 shipped catalog entries have wrong matchMetadata. D1 will surface these as red rows in PLAYTEST_ATTRIBUTION_RESULTS — this is a DELIVERABLE, not a failure. Budget a follow-up boss_encounters.json catalog-fix bead before Codex demo, or the boss-kill lane will silently drop most kills.

8. SessionEventLog's FlushLocked serializes inside the lock (documented tradeoff in D2 step 2 note d). At 5000-entry ring cap under frequent writes, this can stall ObservePlayerName / Record callers if the writer thread is slow. Acceptable for v0.37 ship but budget a lock-narrowing follow-up if profiling shows contention.

9. A2 renders all events as DOM nodes with no virtualization — 5000 entries + textContent generation per render (fired on every chip toggle) may jank on lower-end hardware. Consider adding a virtualization pass or debouncing render() before public ship.

10. The character-name stability gate (N=5 ticks) means the codex opens a file only after ~5 WorldTick intervals of stable PlayerName. Any events emitted during the pre-open window are silently dropped by SessionEventLog.Record (returns false). This is by design (defeats flicker) but means the first ~5 ticks after login are unrecorded — document this in release notes so users don't file 'first kill after login missing' bugs.
