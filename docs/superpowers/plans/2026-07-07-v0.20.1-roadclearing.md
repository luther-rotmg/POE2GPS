# v0.20.1 "Roadclearing" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship v0.20.1 as a coherent "roadclearing" release — closes every carry-forward from the v0.20.0 SDD run, unblocks mainland/preview auto-update users, delivers the two v0.20.0-deferred `/map` features (delta entities + path polylines), sands three dashboard rough edges, and consolidates the settings-migration bookkeeping so the next 30 releases don't grow one-shot bools linearly.

**Architecture:** Additive across five subsystems (SseChannel + Poe2Live/Poe2Atlas + AutoUpdater + browser client + dashboard + RadarSettings). Every new feature is opt-in with a conservative default. The wire format on `/stream` gains optional fields (`paths`, per-entity delta encoding) — old clients keep working. Settings backward-compat is handled by a one-shot migrator that reads old one-shot bools and populates a new `AppliedMigrations: List<string>`, then omits the old fields on next serialize.

**Tech Stack:** .NET 10, C#, x64, Windows. `System.Net.HttpListener` (unchanged from v0.20.0). Vanilla ES2020 + Canvas 2D on the client (unchanged). xUnit for tests. No new NuGet dependencies in production projects.

## Global Constraints

Every task's requirements implicitly include these — no exceptions:

- **Strictly READ-ONLY of the game.** No new memory reads. Vitals re-validation refreshes the shipped `Poe2Offsets.Life`/`Mana`/`ES` constants only.

Path polylines: `/api/paths` route registers only when `EnableWebMap || EnableWebObs`. RC channel: `UpdateChannel` defaults to `"stable"` — behavior identical to v0.20.0 for users who don't opt in.
- **Backward-compatible settings.** A v0.20.0 `config/radar_settings.json` on disk must deserialize cleanly under v0.20.1 code, migrate its 11 one-shot bool fields into `AppliedMigrations: List<string>` on first save, and never lose seeded state. Test on a real v0.20.0 settings file (checked into `tests/POE2Radar.Tests/fixtures/`).
- **No new dedicated threads.** All new async work goes on the ThreadPool via `Task.Run` / `Timer`.
- **No new NuGet dependencies in production projects** (`POE2Radar.Core`, `POE2Radar.Overlay`). `POE2Radar.Tests` unchanged.
- **Version → `0.20.1`** in `src/POE2Radar.Overlay/POE2Radar.Overlay.csproj` (bumped in the final task).
- **CI gates unchanged:** `scripts/compliance-gate.ps1` + `scripts/scrub-strings.ps1 -SelfTest` + full xUnit suite green before RC tag.
- **Wire format is additive.** New `paths` field on SSE and per-entity delta encoding never break a v0.20.0 client. Old client silently ignores unknown fields; new client tolerates missing `paths`.
- **v0.21 "Contributor Flywheel" items (Short List #10-#16) are OUT OF SCOPE** for this plan. Do not fold in Contribute buttons, Worker hardening, contributor receipts, or issue templates. Track for the next release.

## Task 2: SseChannel heartbeat Add/Teardown race fix

**Files:**
- Modify: `src/POE2Radar.Overlay/Web/SseChannel.cs` (`EnsureHeartbeat`, `TeardownHeartbeat`)
- Modify: `tests/POE2Radar.Tests/Web/SseChannelTests.cs` (add race test)

**Interfaces:**
- Consumes: T2 (v0.20.0)'s `SseChannel` class + `Publish`, `AddSubscriberWithSeed`, `RemoveSubscriber`.
- Produces: `EnsureHeartbeat` and `TeardownHeartbeat` guarded so the two paths cannot interleave into a stuck state where a subscriber exists but no heartbeat runs.

**Race being fixed:** If Thread A hits `TeardownHeartbeat` (having just removed the last subscriber) between Thread B's `AddSubscriberWithSeed` `_subs[...] = sub` and B's `EnsureHeartbeat`, then B observes `_heartbeat != null`, returns early, and A's `Interlocked.Exchange(ref _heartbeat, null)` disposes the timer. B's subscriber is left connected with no heartbeat.

**Fix pattern:** gate both `EnsureHeartbeat` and `TeardownHeartbeat` on the existing `_latestLock` object. Cheap because both are called rarely (subscribe / final unsubscribe), not on the hot Publish path.

- [ ] **Step 1: Write the failing test**

Append to `tests/POE2Radar.Tests/Web/SseChannelTests.cs`:

```csharp
    [Fact]
    public async Task Heartbeat_survives_add_teardown_race()
    {
        // Reproducer for the T3 plan-mandated race: while one thread teardowns
        // heartbeat on last-remove, another thread adds a subscriber and calls
        // EnsureHeartbeat. Fix: both paths gate on _latestLock, so the interleave
        // is impossible.
        using var c = new SseChannel();
        var iterations = 500;
        var addCount = 0;
        var removeCount = 0;

        var addTask = Task.Run(() =>
        {
            for (var i = 0; i < iterations; i++)
            {
                var sink = new RecordingSink();
                var id = c.AddSubscriber(sink);
                Interlocked.Increment(ref addCount);
                Thread.SpinWait(50);
                c.RemoveSubscriber(id);
                Interlocked.Increment(ref removeCount);
            }
        });

        var pubTask = Task.Run(async () =>
        {
            for (var i = 0; i < iterations * 2; i++)
            {
                c.Publish(MakeState());
                await Task.Delay(1);
            }
        });

        await Task.WhenAll(addTask, pubTask);
        await Task.Delay(50);

        // Invariant: when _subs is empty, _heartbeat must be null. When non-empty, non-null.
        Assert.Equal(iterations, addCount);
        Assert.Equal(iterations, removeCount);
        Assert.True(c.IsEmpty);
        Assert.Null(c.PeekHeartbeat());
    }
```


Expected: intermittent PASS / FAIL. On a fast machine the test may go 500 iterations without hitting the race — the race window is narrow. To force reliable failure, temporarily comment out the fix instructions (if implemented) or run with 10 000 iterations. For a green baseline before the fix, this test may pass; it acts as regression protection after the fix lands.

- [ ] **Step 3: Apply the fix**

In `src/POE2Radar.Overlay/Web/SseChannel.cs`, locate `EnsureHeartbeat` and `TeardownHeartbeat`. Wrap both bodies in `lock (_latestLock)`:

```csharp
void EnsureHeartbeat()
{
    lock (_latestLock)
    {
        if (_heartbeat != null) return;
        _heartbeat = new Timer(_ => Heartbeat(), null, HeartbeatSeconds * 1000, HeartbeatSeconds * 1000);
    }
}

void TeardownHeartbeat()
{
    lock (_latestLock)
    {
        var t = _heartbeat;
        _heartbeat = null;
        t?.Dispose();
    }
}
```

Rationale: `_latestLock` already exists and is only contended by `Publish` (which locks it briefly to update `_latest`). Add/Remove of subscribers is rare (per-connect / per-disconnect); Publish contention with them is fine. The lock eliminates the check-then-act race that the `Interlocked.CompareExchange` pattern permitted.

```csharp
    /// Filter slot/self-pointer pairs, keeping only slots whose Self field points back at
    /// themselves (drops recycled slots whose Self field is stale from the previous occupant).
    /// Extracted for testability — the production path applies the same check inline in the

        return kept;
```

```csharp
    static object SnapshotForBrowser(RadarState state, EntityDeltaState delta)
    {
        var current = SelectEntitiesRaw(state).ToList(); // (id, x, y, cat, rar, hp, hpMax)
        object entitiesField;
        bool full;

        if (!delta.SeededFullSnapshot)
        {
            full = true;
            entitiesField = current.Select(e => new { e.id, e.x, e.y, e.cat, e.rar, e.hp, e.hpMax });
            delta.LastSent.Clear();
            foreach (var e in current) delta.LastSent[e.id] = (e.x, e.y);
            delta.SeededFullSnapshot = true;
        }
        else
        {
            const float EPS = 0.01f;
            var add = new List<object>();
            var upd = new List<object>();
            var seen = new HashSet<int>();
            foreach (var e in current)
            {
                seen.Add(e.id);
                if (!delta.LastSent.TryGetValue(e.id, out var prev))
                {
                    add.Add(new { e.id, e.x, e.y, e.cat, e.rar, e.hp, e.hpMax });
                    delta.LastSent[e.id] = (e.x, e.y);
                }
                else if (Math.Abs(prev.x - e.x) > EPS || Math.Abs(prev.y - e.y) > EPS)
                {
                    upd.Add(new { e.id, e.x, e.y });
                    delta.LastSent[e.id] = (e.x, e.y);
                }
            }
            var del = new List<int>();
            foreach (var id in delta.LastSent.Keys.ToList())
            {
                if (!seen.Contains(id))
                {
                    del.Add(id);
                    delta.LastSent.Remove(id);
                }
            }
            full = false;
            entitiesField = new { add, upd, del };
        }

        return new
        {
            t = DateTimeOffset.UtcNow.ToUnixTimeMilliseconds(),
            area = state.AreaHash.ToString("x"),
            player = /* ...existing player projection unchanged... */,
            full,
            entities = full ? (object?)entitiesField : null,
            entitiesDelta = full ? null : entitiesField,
            monoliths = SelectMonoliths(state),
        };
    }
```

```csharp
/// <summary>
/// Migrates legacy one-shot bool fields (v0.20.0 and earlier) into RadarSettings.AppliedMigrations
/// on load. Once the migration runs, subsequent serializations omit the legacy fields entirely.
/// </summary>
public static class SettingsMigrator
{
    // Enumerate every legacy one-shot bool and its migration key.
    // Add rows here whenever a new one-shot bool ships and gets consolidated.
    static readonly (string LegacyKey, string MigrationKey)[] Map = new[]
    {
        ("AtlasArrowsSeeded",         "seed:atlas-arrows"),
        ("SeedLandmarksOnce",         "seed:landmarks"),
        ("EntityArrowsSeeded",        "seed:entity-arrows"),
        ("SeedDashboardTutorialShown","seed:dashboard-tutorial"),
        // ...enumerate all 11 legacy keys per Step 1 output...
    };

    public static RadarSettings Migrate(JsonDocument doc)
    {
        var settings = JsonSerializer.Deserialize<RadarSettings>(doc.RootElement.GetRawText(),
            new JsonSerializerOptions { PropertyNameCaseInsensitive = true })
            ?? new RadarSettings();

        var applied = new HashSet<string>(settings.AppliedMigrations ?? new List<string>());
        foreach (var (legacyKey, migrationKey) in Map)
        {
            if (doc.RootElement.TryGetProperty(legacyKey, out var el)
                && el.ValueKind == JsonValueKind.True)
            {
                applied.Add(migrationKey);
            }
        }
        settings.AppliedMigrations = new List<string>(applied);
        return settings;
    }
}
```
