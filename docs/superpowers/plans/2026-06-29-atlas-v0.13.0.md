# Atlas QoL (v0.13.0) — Sikaka v0.16.0 Port — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Port the 7 compliant atlas QoL sub-features from upstream Sikaka v0.16.0 (commit `2f584eb`, on the fetched `sikaka` remote) into POE2GPS, excluding all currency-exchange/pricing parts. Read-only, zero new offsets.

**Architecture:** Vendor 2 new self-contained classes + 2 JSON data tables + 15 PNG assets from `sikaka/main`, then wire them into our (slightly diverged) atlas read/render/config/UI path. The static `AtlasMapData` layer enriches each live atlas node with name/type/group/tags; the rest are filters, a preset seed, colour groups, route chevrons, and on-node content icons.

**Tech Stack:** C#/.NET 10 (x64), Vortice.Direct2D1 (+ Vortice.WIC, already bundled), embedded JSON + PNG resources, xUnit (Core-only).
