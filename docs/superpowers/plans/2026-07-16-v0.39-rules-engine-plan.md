# v0.39 "Rules Engine" — Implementation Plan (draft)

**Status:** DRAFT — awaiting LO approval of parent arc design at `scratchpad/v0.39-rules-engine-arc-design.md`.

**Goal:** Ship a unified `when: selector → then: effect[]` rule engine that layers ON TOP of the existing 6-8 rule surfaces without breaking any of them. Users migrate rules opt-in per-rule via new dashboard buttons.

**Architecture:** New `POE2Radar.Core.Rules` namespace with `RulesFile` record + `RulesFileStore` (mirrors v0.38 F1's `UserPaletteStore` pattern) + `RuleEngine.Compile` producing a pre-indexed `CompiledRuleSet`. Renderer + `ItemFilterEngine` both consume `CompiledRuleSet` via a single injection point. Dashboard gets a new `Rules` tab; each legacy tab gets a "Migrate this rule →" button. All persistence via `config/rules.json` (whole-file atomic-rename envelope, NOT JSONL — rule sets are small).

**Tech stack:** C# 12 records + `System.Text.Json` polymorphism for effects (JsonPolymorphic like v0.37's `CodexEvent`); plain xUnit tests (no FluentAssertions); dashboard.html/css/js edits like v0.38 (worker beads). Target `net10.0-windows`.

## 🚨 Locked Architectural Decisions (READ BEFORE ANY BEAD DISPATCH)

**1. Rule shape:**
```json
{
  "schemaVersion": 1,
  "id": "uuid-v4",
  "name": "user-facing label",
  "priority": 100,
  "enabled": true,
  "when": { "rarity": "unique", "zoneCode": "^delirium" },
  "then": [ { "kind": "ring", "color": "#ff0000" }, { "kind": "pulse", "speed": "fast" } ]
}
```
Higher `priority` wins on conflict. `id` UUID assigned at Save (frontier or worker generates it — never trust client-supplied IDs).

**2. Selector predicates (v1 — flat AND, no nesting):**
| Key | Type | Match against |
|---|---|---|
| `metadata` | regex string | `Entity.Metadata` path |
| `token` | regex string | short entity token (last path segment) |
| `rarity` | enum: `unique\|rare\|magic\|normal` | `Entity.Rarity` |
| `zoneCode` | regex string | current `WorldSnapshot.AreaCode` |
| `inHideout` | bool | `snap.IsInHideout` |
| `minLevel` / `maxLevel` | int | `Entity.Level` inclusive bounds |
| `hasBuff` | regex string | any current buff name matches |

All predicates default absent → doesn't apply. Empty selector `{}` matches EVERYTHING (dashboard shows a warning chip).

**3. Effects (v1 — polymorphic list):**
```
hide          → force-hidden regardless of legacy visibility
tint(#hex)    → override entity color
ring(#hex)    → draw outline (renderer-only, doesn't affect ItemFilterEngine)
label(text)   → override displayed label; {name}/{level}/{metadata}/{zone} tokens
sound(file)   → play config/sounds/<file>.wav on first-sight (dedup 5s), silent on missing
pulse(speed)  → speed = slow (1Hz) | fast (3Hz)
```
Multiple effects apply independently. Effect order in list is application order.

**4. Storage:** `config/rules.json` — whole-file `{schemaVersion:1, rules:[...]}` envelope; atomic write via `.tmp` + rename. NOT JSONL. Load-on-construct + save-on-`Save()` — no debounced flush needed since Save is user-initiated (not per-tick).

**5. Compilation:** `RuleEngine.Compile(RulesFile) → CompiledRuleSet`. Regex pre-compiled with `RegexOptions.Compiled | RegexOptions.IgnoreCase`. Rules sorted by descending `priority`. Coarse pre-index by `rarity` (dict of `Rarity → List<CompiledRule>`) so per-entity walk skips non-matching rules cheaply. Compile once at load + once per Save. NEVER at per-tick.

**6. Legacy interop:** Every existing rule surface (AffixNameplates / BuffNameplates / WaystoneRedFlags / AutoNavPatterns / AtlasHighlight+Arrow+NavTags / CustomLandmarks / ItemFilterEngine) continues to work byte-identically. New engine's effects apply AFTER legacy — a legacy `hide` is preserved even if a new-engine rule matched. Rules-tab UI shows a hint per-legacy-tab: "You can migrate rule X here."

**7. UI: single new tab, per-rule migration.** No auto-migration. Legacy tab's "Migrate this rule →" button opens the Rules tab prefilled with a suggested `name` + selector + effect derived from the legacy rule. User saves it (getting a new `id`). Legacy source stays. User manually deletes legacy source when confident.

**8. API contract:** `GET /api/rules` (ungated, dashboard consumer), `GET /api/rules/<id>`, `POST /api/rules` (loopback-gated, creates or updates by id — no separate PUT), `DELETE /api/rules/<id>` (loopback-gated). 409 on duplicate `name` at POST-create; 200 on POST-update. Mirrors v0.38 F1's endpoint shape.

**9. Rule cap:** 100 rules max, dashboard warning at 50. Exceeding 100 at POST returns 400. Prevents pathological match-cost regression.

**10. NO worker git/bd operations. Preserve all other behavior. xUnit only (plain Assert.*). No AI attribution.**

## Global constraints

- Read-only preserved (no game injection)
- No telemetry beyond approved surfaces
- Local-first (`config/rules.json`)
- Additive (legacy rule surfaces unchanged)
- No new game-memory reach (all predicates use already-read fields)

## Known risks + mitigations

1. **R2 per-tick match cost regression** — mitigate via rarity pre-index + 100-rule cap. Add xUnit bench that walks 10K entities against 100 rules and asserts <5ms wall time. Ships in R2's test set.
2. **R5 selector-builder UI complexity** — regex text input is escape-hatch for advanced users; default to dropdown-driven for common patterns (rarity + zoneCode). Preview area shows "matches X of last Y entities" live.
3. **R7 migration correctness** — 4 legacy shapes = 4 migration mappings. Ship each as separate sub-task within R7 (R7a-R7d); each mapping documented with a truth table in the spec.
4. **Effect: `sound` first-sight dedup edge case** — dedup key is `(rule.id, entity.tokenOrMetadata)`, TTL 5s. If entity leaves + returns in <5s, no re-sound. If rule id changes (edit-in-place), dedup cache invalidated.
5. **JSON polymorphism for effects** — must use `[property: JsonIgnore]` on the discriminator `Kind` (v0.37 `CodexEvent` lesson). Documented in R1 spec.
