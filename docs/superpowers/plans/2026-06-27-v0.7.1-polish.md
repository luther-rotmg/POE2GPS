# v0.7.1 Patch-Resilience Polish — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Six small, independent polish items that harden the shipped v0.6.0 (patch-resilience) + v0.7.0 (Campaign GPS) — a force-re-scan escape hatch, a masthead health pill, console + branding polish, WorldLoop error-hardening, a conservative dead-offset cleanup, and a CHANGELOG.

**Architecture:** Six independent changes, each touching one distinct surface. No new shared subsystem. Read-only throughout.

**Tech Stack:** .NET 10, C# (net10.0-windows, x64), Vortice.Direct2D1, vanilla-JS dashboard in `DashboardHtml.cs`.

## Global Constraints

- **Strictly read-only.** No `SendInput`/`PostMessage`/`SendMessage`/`keybd_event`/`WriteProcessMemory`/injection; no forbidden symbol names anywhere (incl. comments). `/api/rescan` only resets a slot field — it sends no input and reads no game memory.
- **Loopback-gated writes:** `POST /api/rescan` uses the same host guard as `/api/nav` / `/api/settings` (`IsLoopbackHost` → 403 otherwise).
- **Conservative offset cleanup:** delete ONLY offsets that are BOTH explicitly-`⚠ INVALID`/dead AND have **zero references across `src/`** (grep-verify each before deleting). Keep documented "candidate" leads.
- **Anti-detection unchanged:** the console *window title* stays neutral (`Program.cs`); only the loopback dashboard `<h1>` and the console *body* change.
- **Ships as v0.7.1** (version bump in the final pass). README "supports PoE2 0.5.4" badge unchanged (no offset changes).
- CI green: `compliance-gate.ps1`, `scrub-strings.ps1 -SelfTest`, full xUnit suite. No new unit tests (these are wiring/cleanup/docs; the suite must stay green).

            {
                // A persistently-throwing tick must not freeze stale world data on screen or flood stderr.
