> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix `/map` (latent black-map bug since v0.20.0), fix alert volume slider (silent no-op), give the preload panel a collapse toggle + hide-on-spawn filter, and turn Campaign Probe from a silent internal system into an announced opt-out community-data feature with auto-piggybacked uploads on every Contribute click.

**Architecture:** Six shipped items + one LO-manual Cloudflare Worker deploy step. Client-side fixes for `/map` (hex coercion) and volume slider (wireSettings range branch). Preload collapse mirrors the Threshold-era `MonolithPanelCollapsed` pattern. Preload hide-on-spawn adds a per-catalog `SpawnEntityMetadata` binding + per-tick scan in `WorldTick`. Contribute auto-piggyback is a client-side fire-and-forget from three existing button handlers. Public announcement lands in CHANGELOG + README.

**Tech Stack:** C# / .NET 8+ / net10.0-windows / Direct2D + browser Canvas 2D + SSE + HTTP.

```csharp
namespace POE2Radar.Tests.Web;

/// <summary>
/// Signal — SIG-MAP-FIX. Locks the wire-format contract between /api/map and /stream SSE for
/// the zone identifier. /api/map ships areaHash as a JSON number (uint). /stream ships area as a
/// hex string (uint.ToString("x")). Because the two formats differ, the client MUST coerce one to
/// the other before comparing. This test asserts the invariant on both sides so a future refactor
/// tripping either format loudly fails.
/// </summary>
public class ApiMapAreaHashWireFormatTests
{
    [Fact]
    public void ApiMap_AreaHash_Serializes_As_JsonNumber_AndCoercesToSseHexString()
    {
        // Arrange: a plausible areaHash value.
        const uint areaHash = 0x1234_ABCDu;
        var walkable = new byte[] { 0, 1, 1, 0 };
        var json = TerrainMapPayload.ToJson(walkable, w: 2, h: 2, areaHash);

        using var doc = JsonDocument.Parse(json);
        var areaHashElement = doc.RootElement.GetProperty("areaHash");

        // Assert 1: /api/map emits a JSON number, not a string.
        Assert.Equal(JsonValueKind.Number, areaHashElement.ValueKind);
        Assert.Equal(areaHash, areaHashElement.GetUInt32());

        // Assert 2: The SSE-side hex-string coercion of the number matches areaHash.ToString("x").
        var apiHex = areaHashElement.GetUInt32().ToString("x");
        var sseHex = areaHash.ToString("x");
        Assert.Equal(sseHex, apiHex);
    }
}
```

- [ ] **Step 1: Add the per-tick scan in `WorldTick`**

Find the existing preload-frame build block in `RadarApp.cs` (grep `_preloadFrame` — around line 934 per the earlier audit). After the block builds the initial `PreloadHit` list on zone entry, add a per-tick refresh step that:
1. Reads the current `_entities` set (or `snap.Entities` — whichever is the world-thread's authoritative view).
2. For each `PreloadHit` in `_preloadFrame`:
   - Look up the corresponding `PreloadCatalog.Entry` by the hit's `Label` (or the `PathPrefix` field if the hit carries one — check the current shape).
   - If the entry has a non-null `SpawnEntityMetadata`, scan `_entities` for any entity whose `Metadata` string contains that substring.
   - If found, replace the hit in `_preloadFrame` with a copy that has `Spawned=true`. (Since `PreloadHit` is `readonly record struct`, use the `with` expression: `_preloadFrame[i] = _preloadFrame[i] with { Spawned = true };`.)

Structure:
```csharp
// SIG-PRELOAD-HIDE-ON-SPAWN (v0.23): once a bound entity spawns in the entity list, mark the
// preload hit spawned so the renderer skips it. Only entries with SpawnEntityMetadata participate;
// entries without binding stay visible until zone change.
if (_preloadFrame != null && snap.Entities.Count > 0)
{
    for (int i = 0; i < _preloadFrame.Count; i++)
    {
        var hit = _preloadFrame[i];
        if (hit.Spawned) continue;                                     // already spawned, skip
        var entry = PreloadCatalog.FindByLabel(hit.Label);             // or FindByPathPrefix, per catalog API
        if (entry?.SpawnEntityMetadata is not { Length: > 0 } needle) continue;
        for (int j = 0; j < snap.Entities.Count; j++)
        {
            if (snap.Entities[j].Metadata.Contains(needle, StringComparison.OrdinalIgnoreCase))
            {
                _preloadFrame[i] = hit with { Spawned = true };
                break;
            }
        }
    }
}
```

Perf note: the double loop is `O(entries × entities)`. With ≤10 preload entries and ≤50 entities typical for the "interesting" set, that's ≤500 substring checks per world tick — acceptable at ~5s WorldTick cadence for preload refresh (share the existing SR-6 slow-refresh gate if the scan gets expensive — currently unnecessary at this scale).
