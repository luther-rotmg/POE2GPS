using System;
using System.Collections.Generic;
using System.Linq;

namespace POE2Radar.Core.Game;

/// <summary>Classifies a PoE2 entity metadata path to a league-mechanic name via the same canonical
/// substrings the display-rule "Mechanics" use. Pure + unit-tested. Directory-qualified terms avoid the
/// bare-"Shrine"/"Strongbox" cosmetic false-positives.</summary>
public static class MechanicPatterns
{
    private static readonly (string Name, string Sub)[] _patterns =
    {
        ("Expedition", "Expedition2/Expedition2Encounter"),
        ("Ritual",     "Ritual"),
        ("Breach",     "Breach"),
        ("Strongbox",  "StrongBoxes"),
        ("Essence",    "Essence"),
        ("Shrine",     "Metadata/Shrines/"),
    };

    /// <summary>Mechanic names in display order.</summary>
    public static IReadOnlyList<string> Names { get; } = _patterns.Select(p => p.Name).ToList();

    /// <summary>The mechanic name for a metadata path, or null if it isn't a known mechanic marker.
    /// Case-insensitive substring match (mirrors DisplayRules mechanic matching).</summary>
    public static string? Classify(string? metadata)
    {
        if (string.IsNullOrEmpty(metadata)) return null;
        foreach (var (name, sub) in _patterns)
            if (metadata.Contains(sub, StringComparison.OrdinalIgnoreCase)) return name;
        return null;
    }
}
```

namespace POE2Radar.Core.Input;

/// <summary>Formats a Windows virtual-key code to a short display label (F6, R, ], …) for the keybind
/// editor. Pure + unit-tested. The overlay only READS these keys — it never sends input.</summary>
public static class KeyNames
{
    public static string Format(int vk) => vk switch
    {
        >= 0x70 and <= 0x7B => "F" + (vk - 0x6F),            // F1..F12
        >= 0x30 and <= 0x39 => ((char)vk).ToString(),         // 0..9
        >= 0x41 and <= 0x5A => ((char)vk).ToString(),         // A..Z
        0xDD => "]", 0xDB => "[", 0xBA => ";", 0xBF => "/",
        0xBD => "-", 0xBB => "=", 0xC0 => "`", 0xDE => "'", 0xDC => "\\",
        0xBC => ",", 0xBE => ".", 0x20 => "Space",
        _ => "Key" + vk.ToString("X2"),
    };
}
```

### Task 6 — Keybinds dashboard editor + API

**Files:** Modify `ApiServer.cs`, `DashboardHtml.cs`. Consumes Task 2 (`KeyNames.Format`) + Task 5 (`KeybindsSettings`). Verification = build + review.

- **API** (ApiServer.cs):
  - `GET /api/keybinds` (read-only) → an array of `{ action, vk, label, modifier }` for the 9 actions, where `modifier` is `"Ctrl+Alt"` for CycleNext/CyclePrev/NavMenuToggle/SessionReset else `""`, and `label = (modifier.Length>0 ? "Ctrl+Alt+" : "") + KeyNames.Format(vk)`.
  - `POST /api/keybinds` (POST + `IsLoopbackHost`) — body `{ action, vk }`: validate `action` is one of the 9 names and `vk` is in a sane allowed range (F1–F12, A–Z, 0–9, the bracket/punct VKs `KeyNames` handles); **reject a duplicate** (another action already bound to the same vk *and same modifier group*) with 409; else set `_settings.Keybinds.<action> = vk`, `Save()`, return `{ok:true}`.
  - `POST /api/keybinds/reset` (POST + loopback) → `_settings.Keybinds = new KeybindsSettings()`, `Save()`, `{ok:true}`.
- **Dashboard "Keybinds" card** (in Settings): a row per action showing the action name + its current `label`, a **Rebind** button, and a **Reset to defaults** button at the bottom. Rebind flow: on click, the row enters capture mode ("press a key…"); a one-shot `keydown` listener maps `e.code` → VK via a JS map (F1–F12, KeyA–KeyZ, Digit0–9, BracketRight/Left, etc.), ignores bare modifier presses, then `POST /api/keybinds {action, vk}`; on 409 show "already bound"; on ok refresh the list. Load the list via `GET /api/keybinds` on settings load.

JS `e.code`→VK map (provide in the card script): `F1..F12`→`0x70..0x7B`; `KeyA..KeyZ`→`0x41..0x5A`; `Digit0..9`→`0x30..0x39`; `BracketRight`→`0xDD`, `BracketLeft`→`0xDB`, `Semicolon`→`0xBA`, `Slash`→`0xBF`, `Minus`→`0xBD`, `Equal`→`0xBB`, `Backquote`→`0xC0`, `Quote`→`0xDE`, `Backslash`→`0xDC`, `Comma`→`0xBC`, `Period`→`0xBE`, `Space`→`0x20`. Ignore `Control*`/`Alt*`/`Shift*`/`Meta*` codes during capture.

> Implementer note: keep serialization server-side — the dashboard sends `{action, vk}`, never assembles settings JSON. Reuse the existing `$`, `getJSON`, and a `flash` helper. Mirror the `/api/rescan` POST shape for the lean endpoints.
