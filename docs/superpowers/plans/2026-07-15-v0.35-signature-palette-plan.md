# v0.35 "Signature Palette" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Bead-eligible tasks may alternatively be dispatched via `bd create --label worker` + the pipeline; see the **Bead Dispatch** section for the sizing/validate contract.

**Goal:** Ship POE2GPS v0.35 by landing three coherent additive surfaces — an 8-palette signature theme pack (with conformance tests and a swatch preview widget), a stream-safe /obs overlay pipeline (settings round-trip, /obs?mode=safe route, client-side FIFO delay buffer with zone-safe interleave, zone-name mask + hideout coord blur, opt-in entity-name fog), and a themed Session Recap PNG that reads palette hex from the active DashboardPalette via a new Core lookup exposed on /api/settings.

**Architecture:** Three deliverables share the DashboardPalette slug as their single source of truth: the Palette Pack (P1-P3) writes 8 supporter-gated body[data-palette] CSS blocks + options + swatch-preview widget in dashboard.{css,html,js}, all locked by xUnit conformance tests that grep the shipped assets. The Recap PNG (R1-R2) adds POE2Radar.Core.Themes.RecapPaletteMap — a hex mirror of the palette CSS — exposed via a new `paletteColors` field on /api/settings, then rewires saveSessionRecapPng + generateSessionRecap to pull PANEL/ACCENT/TEXT/BORDER from the live settings response with static fallbacks. The Stream-Safe Overlay (S1-S5) adds four RadarSettings.WebObsSafe* fields + /api/settings round-trip, wires /obs?mode=safe through a new AssetHost.SafeObsTransform (rewrites <body> with class="obs safe-mode" plus four data-safe-* attrs), then bootstraps a client-side DelayRingBuffer in map.js that ingests SSE frames FIFO-preserved (leveraging SseChannel's guaranteed full-after-AreaHash-change) and re-clocks on dequeue so state.ring interpolation stays well-formed; zone-mask, hideout-blur, and entity-name-fog helpers layer on top gated by body dataset attrs.

**Tech Stack:** C# .NET 10 (Overlay ApiServer + AssetHost + RadarSettings), POE2Radar.Core class library (Themes/RecapPaletteMap), xUnit test project (POE2Radar.Tests) with TestBoot HttpListener harness, embedded static assets (dashboard.{html,css,js} + map.{html,css,js}) served via AssetHost with ETag suffix keying, browser-side vanilla JS (no framework) inside IIFE.

**Spec:** [docs/superpowers/specs/2026-07-15-gold-tier-4-release-arc-design.md](../specs/2026-07-15-gold-tier-4-release-arc-design.md) v0.35 section.

FULL/DELTA INTERLEAVE — CHOSEN APPROACH (strict FIFO, no drops, no delta promotion):

  * The DelayRingBuffer preserves wire order absolutely. It NEVER drops mid-sequence.
  * SseChannel guarantees a `full=true` frame is the first frame emitted after any zone (AreaHash) change to a given subscriber (SseChannel.cs:84-89 clears LastSent + SeededFullSnapshot on AreaHash mismatch, forcing the next SnapshotForBrowser call down the full path).   * Therefore no delta-to-full promotion is required, and the client never needs to reconstruct a base frame. Deltas dequeued in the middle of a run apply naturally to `state.entities` which was seeded by the earlier dequeued full from the same zone.

ZONE RESET — CHOSEN APPROACH (detected AT POP, not at PUSH):

  * The DelayRingBuffer does NOT inspect frames — zone-reset detection stays in `applyFrameToState` via the existing `if (snap.area !== state.currentArea) onZoneChange(snap.area);` line. The old interpolation ring `state.ring` is flushed inside `onZoneChange` (today's behavior — do not alter).
  * This guarantees: a new-zone position is never rendered until (a) the delay window elapses AND (b) its server-emitted full has been fully applied to state. No lerp across the transition because `state.ring` is cleared inside `onZoneChange` before any new-zone `state.ring.push({t, snap})` inside `applyFrameToState` continues.
  * Consequence: for the duration of the delay window after a zone change, the render pipeline continues to show the previous zone's popped frames (safe — that IS the point of the delay).

RE-CLOCKING (why we overwrite `snap.t` on dequeue): map.js already uses `state.ring` (6-sample interp buffer) with `renderTime = performance.now() + state.serverOffset - RENDER_DELAY_MS`. If we push delayed frames with their original server `t`, `renderTime` will always be > all sample.t and interp `findBracket` returns null, defeating smoothing. Re-clocking each dequeued frame to `now + state.serverOffset` treats each dequeue as "a fresh sample at this instant on the server clock", so the existing lerp window bracketing works unchanged. The 30 s displacement between wire arrival and render is entirely absorbed by the DelayRingBuffer; downstream code stays oblivious.

    state.serverOffset = state.serverOffset === 0

    if (_safeBuf) { _safeBuf.push(snap, performance.now()); return; }
    applyFrameToState(snap);
  }

  function pumpSafeBuffer() {
    if (!_safeBuf) return;
    const now = performance.now();
    const ready = _safeBuf.drainReady(now);
    if (ready.length === 0) return;
    // Re-clock each dequeued frame so state.ring + findBracket + renderTime stay well-formed.
    // See spec DelayRingBuffer note: without this the interp bracket never resolves after 30 s.
    for (const frame of ready) {
      frame.t = now + state.serverOffset;
      applyFrameToState(frame);
    }
  }

## Known Risks & Mitigations

1. R1 <-> P1 SCHEMA MISMATCH: P1 ships palette blocks with 13 CSS vars (--gold, --gold-bright, --gold-deep, --ink, --ink-dim, --ink-faint, --panel, --panel2, --bg, --bg-alt, --line, --line-soft, --good). 2. S4 hideout detection divergence: spec ship-checklist item 4 says `AreaCode prefix match` but S4 implements a monolith/paths/entity-count heuristic with 3-full hysteresis.
