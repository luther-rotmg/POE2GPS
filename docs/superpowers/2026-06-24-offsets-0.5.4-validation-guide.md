# PoE2 0.5.4 Offset Validation — Step-by-Step (for Ryan, in-game)

The code fix is on branch **`fix/offsets-0.5.4`** (already on your machine). These probes
confirm the new offsets read correctly on the live 0.5.4 client and catch any drift beyond
the contained +0x18 `AreaInstance` shift.

## 0. Build (one time)

Terminal at the repo root:

```
dotnet build POE2Radar.slnx -c Release
```

Expect `0 Error(s)` (already verified here). This rebuilds the Research probes with the new
offsets — important, otherwise the probes read the old locations.

## 1. Launch the game

Start PoE2 (0.5.4), log in, and load **into a zone with monsters** (a map is ideal; a busy
town is fine). NOT the login / character-select screen — the probes need to be fully in-game.

## 2. Run the 5 probes

One at a time, in a terminal at the repo root, with the game running. Each attaches to the
live game, prints, and exits:

```
dotnet run --project src\POE2Radar.Research -c Release -- --chain
dotnet run --project src\POE2Radar.Research -c Release -- --info
dotnet run --project src\POE2Radar.Research -c Release -- --vitals
dotnet run --project src\POE2Radar.Research -c Release -- --rarity
dotnet run --project src\POE2Radar.Research -c Release -- --inventory --itemmods
```

## 3. What PASS looks like

| Probe | Read it for | ✅ PASS | Confirms |
|---|---|---|---|
| `--chain` | the `LocalPlayer` line | says **MATCH**, addresses non-zero | chain + `LocalPlayer` 0x5B8 |
| `--info` | area code + level + hash | real area code (e.g. `G1_town`), sane level, non-zero hash | the **low** AreaInstance fields didn't shift |
| `--vitals` | Health / Mana / ES | match your **in-game orbs** (e.g. 980/980 HP) | `Life` component (drifted independently last patch) |
| `--rarity` | entity count + list | sensible count for the zone (dozens in a map), rarities/hostility look right | `AwakeEntities` 0x6D8 + `SleepingEntities` 0x6E8 |
| `--inventory --itemmods` | inventories + item mods | inventories list with box sizes; mods render as text | `ServerData` chain 0x598 + inventory |

## 4. Report back

- **All 5 look right** → reply **"all pass"**. I merge to main, ship **v0.5.1**, post the
  Discord note.
- **Anything looks off** (a probe errors, says "not in game", shows 0 entities, mismatched
  HP, or garbage) → **paste me that probe's console output**. That means something *else*
  drifted in 0.5.4; I'll pinpoint the offset, fix it, and you rebuild + re-run just that one.

## Troubleshooting

- **"Could not resolve chain (in game?)"** → you're at the menu / a loading screen. Get fully
  into a zone and retry.
- The probe must run as the **same Windows user** as the game. If PoE2 runs elevated (admin),
  run the terminal as admin too.
- If `dotnet` isn't found, use the same terminal where the build in Step 0 worked.
