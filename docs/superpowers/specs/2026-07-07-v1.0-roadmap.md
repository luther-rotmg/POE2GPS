## 1. Current State

### Elevator
POE2GPS is a strictly read-only Path of Exile 2 overlay: `PROCESS_VM_READ`-only memory reads, no input injection, no telemetry, no pricing egress, one narrowly-scoped HTTPS update check against `github.com/luther-rotmg/POE2GPS`. Ships as a single self-contained Windows x64 EXE with silent SHA-256-verified auto-update and one-generation rollback. The overlay draws Direct2D on top of the PoE2 window (radar, terrain, HP bars, nameplates, atlas overlay, monolith rewards, session HUD, preload alerts). A loopback HTTP server on `:7777` powers a rich dashboard (`/`), a monitor-refresh browser minimap (`/map`), an OBS Browser Source view (`/obs`), and a 30 Hz SSE snapshot stream (`/stream`). The tree is disciplined by a CI compliance gate that fails the build on any input/write symbol or pricing regression.

### Brand pillars (hard boundaries for every roadmap item)

- **100% read-only.** `PROCESS_VM_READ` only. CI (`scripts/compliance-gate.ps1`) hard-fails on any of ~25 input/write symbols (`SendInput`, `WriteProcessMemory`, `VirtualProtectEx`, `CreateRemoteThread`, `SetWindowsHookEx`, `PostMessage`, etc.) and on `OpenProcess` acquiring write/operation access.
- **No telemetry, no pricing, no phone-home except one update check.** Second CI guard specifically blocks `Pricing/` dir, `PriceBook`, and `/api/prices` reappearing. Reward overlays are names-only, no value chips.
- **Opt-in for anything invasive; conservative defaults.** Nameplates, ground labels, monolith panel, Director, Campaign GPS, God-Roll, Audio, Session HUD, Preload Alert, Discord RP, LAN reads all default OFF. Hidden-from-capture defaults ON.
- **Auto-update ratchet: SHA-256 mandatory, GitHub-only, extracts only `Overlay.exe` + `CHANGELOG.md`, one-generation rollback via `Overlay.old.exe`, crash-loop threshold of 3.** Never touches `config/` or `icons/`, never touches the game.
- **Loopback HTTP + no PII on the wire.** `/api/*` never exposes character name, account, or absolute address. All state-mutating POSTs Host-header gated. Community Contribute is per-click opt-in, sends only entity path → friendly name / label.

## 5. Surprise / Delight

Six items LO didn't ask for that fall out of the shipping data + web-view substrate.

### S1. Session recap PNG generator on `/obs`

**Pitch.** At session-end (dashboard button + optional hotkey), the browser generates a 1920×1080 PNG summarizing kills-by-rarity, maps/hr, deaths, XP-eff, top monolith rewards, most-visited zones — laid out in the POE2 parchment palette with the wordmark. One-click download; drag into Discord, Twitter, Reddit. Substrate: canvas of `/state.session` at end-of-run, no new server code. **Effort:** S. **Certainty:** HIGH. **Why it's a surprise:** LO has been shipping counters for months; nobody thought to make them shareable.

### S2. Atlas heat-map browser view

**Pitch.** `/atlas-heatmap` — a browser page that renders the Atlas node graph coloured by how much time the user has spent on each node type this league (already tracked by `SeenPoiLog` + zone timers). Streamers get a visualization of their farming strategy. Substrate: `/api/atlas` + a per-node accumulator on top of existing zone events. **Effort:** M. **Certainty:** MEDIUM. **Why it's a surprise:** turns a diagnostic ledger (SeenPoiLog) into a keepsake artifact.

### S3. Ambient soundscape mode

**Pitch.** Optional per-zone ambient sound layer driven by zone code (act 1 forest → wind + birds, Utzaal → dry wind + distant chants, mapping T15+ → arcane hum). All pure-tone/short-loop assets bundled with the EXE; the same `AudioCue` plumbing used for alerts. Streamer-friendly, atmospheric, and completely optional. **Effort:** M. **Certainty:** MEDIUM. **Why it's a surprise:** users think the audio system is just alerts; giving it a chill lane is a mood-shift.

### S4. "Where did I die?" death heatmap on `/map`

**Pitch.** Every death event drops a pin (x/y/zone) into a small ring buffer; `/map` view has a toggle chip that renders red circles at the last N death positions, per zone. Zero new offsets — the death event is already surfaced by `SessionTracker`. **Effort:** S. **Certainty:** HIGH. **Why it's a surprise:** self-mocking + genuinely useful for boss-arena awareness. The kind of thing users screenshot with the caption "so THIS is where I keep dying."

### S5. Streamer chat overlay integration (Twitch IRC read-only)

**Pitch.** Optional side-panel widget in `/obs` that renders the last N Twitch chat messages, colour-coded by badge tier, powered by the anon `justinfan*` IRC channel (no OAuth needed, purely read-only). Streamers get one less browser source to layer. Compliance: only reads a *public* IRC stream, no chat sending, no game interaction. **Effort:** M. **Certainty:** MEDIUM. **Why it's a surprise:** POE2GPS becoming a streaming-companion beyond just "here's a browser source."

### S6. Mini-map export → poe2wiki bounty ("this zone's layout, drawn by 40,000 players")

**Pitch.** Users opt-in to submit the walkable-terrain bitmap for a zone (already computed) to the same community pipeline. Server-side average of contributed layouts produces a canonical zone map per area code, served back as `atlas-icons.json`-adjacent data. Every future install starts with better default landmarks. Substrate: `/api/map` payload + Worker + a nightly diff aggregator. **Effort:** M. **Certainty:** LOW (needs careful design of the aggregation math). **Why it's a surprise:** flips the community pipeline from names-only to layout-canonicalization; effectively crowdsources a proper world map, which POE2 has never had.

## 6. v1.0 North Star

### What "feature-complete" looks like

POE2GPS v1.0 is the definitive read-only companion overlay for Path of Exile 2 — the one you install and never think about again because it survives every league patch, never touches the game process, and grows its catalogs from the community's playtime. It ships as a single self-updating EXE with a full browser dashboard, three optional monitor-refresh browser views (map, OBS, tablet co-pilot), a mature Objective Director covering campaign through endgame Uber pinnacles, and a complete data catalog spanning skill gems, uniques, essences, ascendancy nodes, and boss encounters. Its patch-resilience layer — offset auto-heal, AOB fallbacks, community-fed name/label/landmark packs — means a new PoE2 major patch is a shrug, not a scramble.

By v1.0.0, POE2GPS has consumed every reasonable read-only overlay category (routing, navigation, atlas planning, monolith preview, session tracking, gear scoring, price-check via clipboard-only, waystone risk analysis, boss cheat-sheets, campaign speedrun timing, whisper log, replay recording, live pricing snapshot) while remaining honest about its ceiling: a passive read-only overlay that respects the game and the player, positioned in the lowest-risk bucket the medium allows, and paired with the limited-user Windows account posture the README already documents.

### Non-goals (5 things POE2GPS will NEVER do)

- **Never emit input into the game.** No `SendInput`, no `keybd_event`, no `PostMessage`, no auto-flask, no auto-pickup, no "walk to waypoint" mover, no auto-whisper/auto-invite. Compliance gate enforces this at the symbol level.
- **Never write to the game process.** No `WriteProcessMemory`, no hooks, no injection, no byte-patching. `PROCESS_VM_READ`-only forever.
- **Never emit telemetry, pricing egress, or any HTTPS traffic outside the single update check and the opt-in Community Contribute click.** No usage analytics, no crash reporters, no "anonymous" opt-out. No poe.ninja embedding, no Div/Exalt value chips on reward overlays.
- **Never surface character name, account name, or precise player position on any `/api/*` endpoint.** Loopback default; LAN reads opt-in; POSTs Host-header gated.
- **Never rely on an Overwolf-style host, an ads layer, an account wall, or a lock-in installer.** Free, self-contained EXE, silent auto-update, one-generation rollback. That's the whole shape.

### Signals we've hit v1.0

- **Zero pending offset drift on the current PoE2 patch** — vitals, atlas, entity, buff, mod, chest, loaded-files, and quest-memory all read cleanly with `Research --verify` returning green across the board.
- **Community catalogs saturate** — `entity_names.json` covers >99% of encountered monsters in a full campaign+endgame run; landmark, monolith, buff, and preload catalogs all grown from real playtime with named contributor credit blocks in every release CHANGELOG.
- **Objective Director graduates from experimental to default-on** — quest-memory precision layer covers full campaign, and endgame director surfaces atlas-progress-aware targets ranked with community-tuned tiers.
- **Three browser views (`/map`, `/obs`, `/copilot` or `/rules`) are the recommended default install** — dashboard becomes the settings surface, not the primary UX.
- **Patch-day cadence: LO ships an updated release within 6 hours of a PoE2 major patch drop**, with zero manual offset probing thanks to auto-heal + AOB fallback + community-fed drift-correction packs.
