# POE2GPS v0.21 — "Guided Campaign" Design Spec

**Date:** 2026-07-07
**Prior release:** v0.20.1 "Roadclearing" (2026-07-07)
**Base commit:** `main` HEAD post v0.20.1 tag
**Roadmap references:** SL #10, #11, #12, #13, #14, #15, #16 (from [v1.0 roadmap §7.5](2026-07-07-v1.0-roadmap.md)); PMS-4 stubbed; syrairc DM required for tag.

## 1. Goal

Ship a two-track release that:

1. **Track A — EC2 Guided Campaign:** integrates syrairc's ExileCampaigns2 route data + advance-engine logic into POE2GPS as a new step-by-step campaign guide surface, running alongside (not replacing) the existing Campaign GPS + Objective Director.
2. **Track B — Community Pipeline Hardening:** completes the community-contribution rails so buffs and preloads can be submitted through the same funnel as atlas objectives, hardens the Worker against abuse, and closes the divergent-merge-script bug that dropped LO's dogfood pack on the floor in issues #4/#5.

Both tracks tag as **v0.21 "Guided Campaign"** in one drop. The EC2 port is the headline feature; the pipeline work is supporting hardening.

## 2. Non-negotiables

- **Zero memory writes.** Every new memory access is a read of `ServerData` or `IngameState.Data`, following the shipped `PlayerInventories` chain pattern (`Poe2Live.cs:1717+`).
- **Zero-cost-when-off.** `WorldStateAdapter` and `RouteCursor` respect `_settings.EnableCampaignGps`; adapter is constructed unconditionally at startup so runtime toggle can't race lazy init, `Tick` gated on the setting.
- **v0.20 wire-format compat.** All new SSE data lands as **additive** keys on the `/stream` payload. No renames. No removals. New key: `CampaignGuide`.
- **No internal-tooling paths in public surfaces.** CI grep gate rejects `superpowers/`, `.superpowers/`, `docs/superpowers/` in README, CHANGELOG, `.github/ISSUE_TEMPLATE/*.yml`, root `CONTRIBUTING.md`, `docs/community-pipeline.md`, any Worker HTML, any in-app string.
- **syrairc DM hard-gates the tag.** Attribution ships in two phases: DRAFT (sentinel tokens, unblocked now) → FORMALIZE (real license + pinned commit hash, blocks tag).
- **Read-only philosophy preserved.** No input injection, no telemetry, no egress except the existing one-shot update check.

## 4. Architecture

### 4.1 Track A — EC2 Guided Campaign

Two independent Campaign-related systems already ship in POE2GPS:

1. **Cross-zone Campaign GPS** (`Core/Campaign/CampaignGps.cs`) — `Decide` takes zone code + `IQuestProgress.CurrentStep(zone)` + landmarks/entities + player pos, returns a `GpsInstruction`. The only `IQuestProgress` implementation today is `ZoneOrderProgress` (a forward-only ordinal latch, no memory reads).
2. **In-zone Objective Director** (`Core/Campaign/ObjectiveDirector.cs` + `Overlay/Web/CampaignObjectives.cs`) — flat matcher catalog ranked per tick against live entities/landmarks.

Track A adds a **third parallel system**, wired at the same reconciliation point (`RadarApp.CampaignReconcile:2493`):

3. **EC2 Guide** (new `Core/Campaign/Guide/`) — ordered `steps[]` cursor with per-tick `AdvanceEngine` evaluation and area-change forward-snap. Publishes a `CampaignStepInstruction` immutable record struct into a new `_campaignGuide` field parallel to the existing `_campaignGps`.

**Data flow:**

```
world thread (per tick):
  Poe2Live snapshot
    → WorldStateAdapter (implements IWorldState)
      → AdvanceEngine.Tick(worldState)
        → RouteCursor.AdvanceIfSatisfied()
          → _campaignGuide (immutable published snapshot)

SSE payload builder (RadarApp.cs:1500):
  snapshot.CampaignGuide = _campaignGuide         # NEW additive key
  snapshot.CampaignGps   = _campaignGps           # unchanged

Dashboard render:
  campaign panel shows: step text + degradation badge + syrairc attribution
```

### 4.2 Track B — Community Pipeline Hardening

**Worker (Cloudflare):** three sibling routes instead of a discriminated-union payload schema.

- `POST /submit-atlas` — existing `{names, objectives}` shape.
- `POST /submit-buffs` — buff metadata paths + tier.
- `POST /submit-preload` — metadata paths only, reject bare `.dds`/`.ao`.

All three share middleware: NFKD-normalized leet-fold profanity filter (drops the 8/12-char length gates), simple KV counter rate limit (5 requests / 60s per `CF-Connecting-IP` — **not** Cloudflare's `unsafe.bindings` native Ratelimit binding, which is experimental), gh dispatch. `buildIssue` emits type-specific title + sub-label (`community-pack` + one of `buffs`/`preload`/none).

**Ordering (hard):** Worker deploys to prod FIRST (verified via smoke script), THEN CF-DASH-BUTTONS ships in the app build, THEN tag. Stale desktop clients hitting new routes before deploy see a clean 404 (unambiguous) rather than a schema-mismatch 400.

## 6. Quest-flag strategy (LOCKED)

Six stubbed `WorldStateAdapter` methods, all inlined as private methods (no `IQuestFlags` sub-interface):

```csharp
private bool QuestFlagSatisfied(Pattern p)   => false;   // PMS-4 blocks
private bool WaypointPulsed()                => false;   // PMS-4 blocks
private int  SatisfiedFlagCount()            => 0;       // PMS-4 blocks
private int  TalkProgress(Pattern p)         => 0;       // NpcDialog offset chain not yet recon'd
private int  InteractProgress(Pattern p)     => 0;       // Targetable byte reader only exists in Research/Program.cs
private bool LootSatisfied(QuestItem q)      => false;   // PlayerInventories walks CARRY only; quest-inventory bucket not recon'd
```

**Graceful degradation path** (same pattern EC2 itself uses):

- Live signals (Kill / InAreaSatisfied / ProximitySatisfied / player-inventory Loot) advance normally in real time.
- Steps whose *only* advance signal is a stubbed method stall until player crosses into next zone.
- `RouteCursor` area-change forward-snap moves cursor past stalled step at zone boundary.
- Persistent top-of-panel badge: `Some steps require v0.22's quest-flag reader — they'll advance at zone boundary until then.`
- Themed CHANGELOG entry release-notes this as designed behavior so users don't file it as a bug.

**v0.22 unlock path (PMS-4 extended scope):**

- Reverse-engineer FOUR offset chains in ONE Research probe session on the same play window: `ServerData.QuestFlagsVec`, `NpcDialog` root, `Targetable` component byte layout, quest-inventory bucket enumeration.
- Add accessors to `Poe2Live.cs` mirroring the shipped `PlayerInventories` chain pattern.
- Swap-in for all six stubbed methods = six body-only edits inside `WorldStateAdapter.cs`. ZERO changes to `AdvanceEngine`, `RouteCursor`, `CampaignReconcile`, or SSE payload shape.

## 8. Risks

1. **syrairc DM gate.** Verbal permission unblocks all IMPLEMENTATION tasks. DM formalizing license + pinned commit hash HARD-BLOCKS the tag. Multi-day variance PMS same category as PMS-3.
2. **602 KB route.json embedded resource.** 4-5× largest existing embedded json. Release zip stays under GitHub asset cap. Themed release notes should mention the size delta.
3. **EC2 port coupling risk.** Recon suggests `IWorldState` seam is clean, but no ported file has been independently opened. If `EC2-PORT` surfaces ExileCore2 references once files open, task splits into `port` + `strip-couplings`. Certainty drops to MEDIUM until first file opened. Mid-execution checkpoint planned.
4. **Worker deploy ordering.** `CF-WORKER` MUST land in prod (smoke-verified) BEFORE `CF-DASH-BUTTONS` PR opens. Stale desktop clients hitting new routes before deploy get a clean 404 (unambiguous).
5. **SL #9 `TieredCatalog<T>` shipping assumption.** Confirm shipped in v0.20 before starting `CF-DASH-BUTTONS`. If not shipped, DEFER SL #10 buff-fold to v0.22; SL #11 preload-fold is independent.
6. **v0.20 wire-format compat.** New `CampaignGuide` SSE field is additive-only. Verify gate = v0.20.1 client sniff transcript + golden DTO snapshot test. Regression here silently breaks external overlay clients after users update backend.
7. **Six stubbed WorldStateAdapter signals.** Release-note as graceful degradation, not bug. Users will notice steps stalling at flag-only / talk-only / interact-only / quest-item-only advance points. Badge + CHANGELOG must preempt bug reports.
8. **Label divergence (SL #12 + #16).** Existing OPEN atlas-submission issues need `relabel-atlas-issues.sh` run in production BEFORE `CF-DEPRECATE-ATLAS` lands — otherwise they get orphaned.
9. **Attribution placeholder leak.** `TODO(syrairc-*)` sentinels sit in the branch between DRAFT completion and FORMALIZE. CI grep gate must be wired BEFORE the DRAFT PR merges. Tag-time gate rejects any surviving sentinel.
10. **`WorldStateAdapter` runtime toggle race.** Adapter constructed unconditionally at startup even when `EnableCampaignGps=false`. Small always-on allocation footprint (one object graph) prevents the lazy-construction race.
11. **`CF-MERGE-RECEIPT` idempotency.** Script prints credit block on stdout; mid-loop crash re-runs from scratch. Fold-time dedup covers double-fold; credit block is deterministic (unique-sort by handle) so re-runs produce identical output.
12. **Internal-tooling leak audit.** CI grep gate on every task diff touching a public surface.
