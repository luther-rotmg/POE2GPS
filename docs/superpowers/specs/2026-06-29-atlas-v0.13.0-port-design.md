# v0.13.0 — Atlas QoL (Sikaka v0.16.0 port) — Design / Port Spec

**Date:** 2026-06-29
**Branch:** `feat/v0.13.0-atlas-qol`
**Status:** Approved scope (user: "do everything in all the tiers") — spec for implementation

## Goal

Port the **7 compliant atlas QoL sub-features** from upstream Sikaka v0.16.0 (commit `2f584eb`) into
POE2GPS, **excluding** the bundled Currency-Exchange / pricing reader and its dependencies. All read-only,
**zero new offsets** required (confirmed by the port-scope analysis).

## Compliance invariant (non-negotiable)

Strictly read-only. The atlas features only read already-validated atlas node memory + embedded static
JSON, and draw. **EXCLUDE entirely** (the bundled non-compliant additions):

- `src/POE2Radar.Core/Game/Poe2CurrencyExchange.cs` (Kalguur order-book reader)
- `Poe2Offsets.CurrencyExchange`, `Poe2Offsets.MouseOver`, `Poe2Offsets.GroundLabels.ContainerFlagFingerprints`
- `Poe2Live.MouseOverEntity()`, `ResolveGroundLabelContainer()`, `WalkGroundLabels()`, and the pricing-scoped
  `ScanLootLabels()` refactor (keep our current `ScanLootLabels`)
- `RadarSettings.CurrencyExchange` + `CurrencyExchangeSettings`; all `Exchange*` `RenderContext`/`RadarApp`
  fields + `ExchangeRow` + `UpdateCurrencyExchange()` + `OverlayRenderer.DrawCurrencyExchange()` + `ColExchange*`
- `ApiServer` `currencyExchange` `/state` object + patch case + `TryParseCurrencyExchange()`; any exchange dashboard UI
- All Research `--exchange*`/`--mouseover`/`--loot*` probes (Research isn't in POE2GPS anyway)

The compliance gate (`scripts/compliance-gate.ps1`) + scrub stay green. No input emission, no pricing,
no process writes. README badge stays `0.5.4`.

## Vendoring (from the `sikaka` remote, already fetched)

New files/assets are pulled from Sikaka and adapted, not hand-rewritten:
- Text files: `git show sikaka/main:<path> > <our path>` (then namespace/type-adapt as needed).
- Binary assets (the 15 PNGs): `git show sikaka/main:<path> > <path>` works for binary in Git Bash, or
  `git checkout sikaka/main -- <path>` then `git mv` into our tree.
- Our `Poe2Atlas.cs` matched Sikaka's pre-v0.16.0 signature, so the read-side hooks graft cleanly.

## The 7 features (tiered by value-to-risk; build in this order)

### Tier 1 — foundational data layer (no risk)
**#1 `AtlasMapData` + `atlas_maps.json`** — per-node display name / map type (`normal`/`unique`) / group /
tags (`lineage`, `arbiter`) from GGG data. Vendor `src/POE2Radar.Core/Game/AtlasMapData.cs` + `atlas_maps.json`
(embedded resource). Widen `Poe2Atlas._tagCache` tuple + `ResolveTags()` to carry `MapMeta`; add 3 params to
`AtlasNodeLive` (`MapType`, `MapGroup`, `MapDataTags`) with safe defaults. **Effort S.** *Prerequisite for #2/#5/#6/#7.*

**#2 `atlas_content.json` + content descriptions** — content-id → {name, icon, desc}; enriches the dashboard
`allTags` projection with `desc`/`icon`. Piggybacks #1 (same `AtlasMapData`). **Effort S.**

**#3 Hide completed / accessible filters** — `RadarSettings.AtlasHideCompleted=true`, `AtlasHideAccessible=false`;
two `if/continue` guards in `BuildAtlasMarks`; extend the `_atlasLastSig` change-hash; two dashboard checkboxes +
two API patch cases. `AtlasNodeLive.Completed`/`.Accessible` already exist. **Effort S.**

### Tier 2 — high-visibility QoL
**#6 Built-in "Map Targets" preset** — static `BuiltInAtlasTargets[]` (Citadels/Halls/uniques by code) +
`DefaultAtlasGroups[]`; `SeedAtlasDefaults(nodes)` seeds them once into the existing `AtlasHighlightTags`/
`AtlasHighlightColors`, gated by a new `RadarSettings.AtlasTargetsSeeded`. Additive, never clears user rules.
**Effort S.**

**#7 Colour groups editor** — `RadarSettings.AtlasGroups` (`List<AtlasMapGroup>`) + `AtlasGroupsSeeded`;
`BuildAtlasMarks` match logic also checks `MapDataTags`/`MapType` and falls back to a group colour; `allDataTags`
added to the API state; dashboard colour-group editor (add/edit/remove). API `atlasGroups` patch + `TryParseAtlasGroups`.
Requires #1 + #6. **Effort M** (dashboard editor is the heavy part).

### Tier 3 — render polish
**#4 Route chevrons + off-screen culling fix** — `OverlayRenderer.DrawAtlasRoute()` + the `DrawAtlas()` auto-route
loop gain directional arrowhead chevrons (`DrawAtlasChevrons`), per-edge phase interleaving (`AtlasEdgeKey`), and an
on-screen `On(p,margin)` gate that fixes the off-screen-relPos route-jitter bug. `RadarSettings.AtlasRouteArrowSpacing=8f`
+ `RenderContext` param + API patch. **The culling fix is a correctness win on its own. Effort M.**

**#5 On-node PNG content icons (`AtlasIconCache`)** — vendor `src/POE2Radar.Overlay/Overlay/AtlasIconCache.cs`
(WIC PNG decode → lazy `ID2D1Bitmap`) + 15 `src/POE2Radar.Overlay/Assets/AtlasIcons/*.png` (embedded resources).
`AtlasMark` gains `ContentIcons`/`Visible`; `OverlayRenderer` gets `_atlasIcons` + `DrawAtlasContentIcons()` (drawn
only on fogged `!Visible` nodes, since the game draws its own on revealed ones) + dispose. `RadarSettings.AtlasShowContentIcons=true`
+ `AtlasContentIconSize=26f` + two `RenderContext` params + two API patches + two dashboard controls. Requires #1+#2.
`Vortice.WIC` ships inside the already-referenced `Vortice.Direct2D1` package — no new NuGet. **Effort M.**

## Offsets

**Zero new offsets.** The compliant features use only embedded JSON + already-live-read atlas node fields
(`Completed`/`Accessible`/`Tags`/`Visible`/`MapCode`) + PNG bitmaps. No `--atlas-probe` re-validation needed.

## Architecture / boundaries

- **Core:** `Game/AtlasMapData.cs` (new), `Game/atlas_maps.json` + `Game/atlas_content.json` (new embedded),
  `Game/Poe2Atlas.cs` (`AtlasNodeLive` + `ResolveTags`/`_tagCache` widen). No Core→Vortice.
- **Overlay:** `Overlay/AtlasIconCache.cs` (new) + `Assets/AtlasIcons/*.png` (new), `Overlay/OverlayRenderer.cs`
  (chevrons + content icons), `Overlay/RenderContext.cs` (`AtlasMark` fields + new ctx params), `RadarApp.cs`
  (filters, seed, group colour, content-icon resolution), `Config/RadarSettings.cs` (new atlas settings),
  `Web/ApiServer.cs` (new patch cases + `allDataTags`), `Web/DashboardHtml.cs` (filters, colour-groups editor,
  icon controls).
- `*.csproj`: 2 embedded JSON in Core, the PNG glob in Overlay.

## Testing

- **Core unit test:** `AtlasMapData` load + `Get`/`TryGet`/`ContentDesc`/`ContentIcon` resolution from the embedded
  JSON (pure, Core-only project).
- **Build + live (atlas open):** each render/dashboard feature is behavior-additive — verified by 0/0 build +
  full suite + compliance/scrub gates + per-task & final review, then a live atlas smoke (the new filters/chevrons/
  icons/targets/groups behave; nothing pricing-related appears).

## Out of scope (YAGNI)

- The Currency-Exchange reader and everything pricing (excluded above).
- The `MouseOverEntity`/ground-label-container read (no compliant consumer in POE2GPS).
- Any feature beyond the 7 listed.
