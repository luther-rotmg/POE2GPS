# /map + /obs — 30 Hz Push / 60 Hz Draw, In-Game Clone (v0.20.0) — Design

**Date:** 2026-07-06
**Status:** approved (design), pending spec review
**Release:** v0.20.0 (browser-view rework: SSE push at 30 Hz, `requestAnimationFrame` draw at monitor refresh, in-game POE2 visual language, feature-gated + opt-in). Includes Tencent CN client compatibility recon (appendix, doc only).

## Goal

Two feature threads, one release:

1. **/map + /obs upgrade.** Make the browser views feel like a live in-game minimap in a browser tab: perceptual 60 Hz smoothness (or higher on high-refresh monitors), in-game POE2 visual language, and the full data set the native Direct2D overlay already knows (atlas nodes, monoliths, landmarks, fog of war). Transport pushes at 30 Hz — the "60 Hz" in the title refers to client render cadence, not push rate. Both views are opt-in via `RadarSettings` and default OFF, so users who don't want them incur zero runtime cost.
2. **Tencent CN client — recon only.** Ship a design brief that maps every blocker to CN client compatibility with file-and-line citations, high-level costs to unlock, and a clean "don't commit code time yet" recommendation. No CN plumbing in this release. Documented, not built. The one CN-adjacent piece worth doing (configurable `UpdateUrl`) is explicitly deferred to v0.20.1 to keep this release focused.

## Background

`/map` and `/obs` are the two localhost browser surfaces exposed by [`ApiServer`](../../../src/POE2Radar.Overlay/Web/ApiServer.cs). Today both are served from a single-file inline HTML/JS blob at [`MapPageHtml.cs`](../../../src/POE2Radar.Overlay/Web/MapPageHtml.cs), which does:

- `setInterval(tick, 1000)` — hard **1 Hz** browser poll.
- Three serial `await fetch()` calls per tick (`/state`, then `/api/map` on `areaHash` change, then `/entities?limit=600`), then one full canvas redraw.
- No SSE, no WebSocket, no `requestAnimationFrame`, no interpolation. Grep confirms zero `WebSocket|EventSource|SSE` matches anywhere in `src/POE2Radar.Overlay/Web`.

Meanwhile the server memory-reads at **30 Hz** (`WorldHz = 30` in [`RadarApp.cs:28`](../../../src/POE2Radar.Overlay/RadarApp.cs)) and re-publishes a fresh `_state` snapshot at the end of each `Tick()`. The browser throws away 29 of 30 samples every second.

Visually the browser view is bare — flat walkable cells, colored circles, a centered green circle for the player. The **native Direct2D overlay** already renders terrain edges, atlas nodes with icons, monoliths with reward text, landmarks, A* paths, HP bars, and off-screen arrows ([`OverlayRenderer.cs:1257-1348`](../../../src/POE2Radar.Overlay/Overlay/OverlayRenderer.cs), [`TerrainBitmap.BuildFrom`](../../../src/POE2Radar.Overlay/Overlay/TerrainBitmap.cs)). The browser just isn't consuming the endpoints or state fields that already exist (`/api/atlas`, `/landmarks`, `_state.Monoliths`).

The bottleneck is transport + client draw cadence, not the renderer, not the underlying data. Server has fresh state at 30 Hz — the browser needs a push channel and a real draw loop to consume it.

## Architecture (2–3 sentences)

A new `SseChannel` fans out `_state` snapshots at 30 Hz to any subscribed browser via a new `/stream` Server-Sent Events endpoint. The client keeps a 6-slot ring of `(serverT, snapshot)` samples, computes a smoothed `serverOffset` (from server clock to `performance.now()`) on connect and rolls an EMA on it, then draws every `requestAnimationFrame` from an interpolated pose (`RENDER_DELAY_MS = 66.67`, two sample intervals behind the newest) and applies an in-game POE2 skin — dark terrain interior with warm off-white edge outlines, client-inferred fog of war, atlas/landmark/monolith icons over data the server already publishes. Two new settings (`EnableWebMap`, `EnableWebObs`, both default `false`) gate route registration and channel instantiation so the code path is zero-cost when unused.

## Tech Stack

.NET 10, C#, x64, Windows. `System.Net.HttpListener` (async, multithreaded via `GetContextAsync + Task.Run`, chunked-encoded SSE with `SendChunked = true`). Vanilla ES2020 + Canvas 2D on the browser, no framework, no bundler, no npm. Assets embedded as C# managed resources in `POE2Radar.Overlay.dll`. xUnit for unit + integration tests. **No new NuGet dependencies in the production projects** (`POE2Radar.Core`, `POE2Radar.Overlay`); `POE2Radar.Tests` may add `Microsoft.Extensions.TimeProvider.Testing` for the heartbeat timer test — this is a test-only package and does not ship in the released binary.

## Global Constraints (binding)

- **Strictly READ-ONLY of the game.** No new memory reads. All data added to the browser view is already read by the world thread today.
- **Both browser views default OFF.** Fresh installs have `EnableWebMap = false` and `EnableWebObs = false`. Users opt in via the dashboard. On upgrade from v0.19.6, missing keys deserialize to `false` (the same as fresh installs) — existing users see `/map` return 404 until they opt in. No migration prompt.
- **Zero-cost-when-off contract.** With both toggles off: no `SseChannel` allocation, no `AssetHost` allocation, no `/map` / `/obs` / `/stream` / `/api/map` / `/api/atlas` / `/landmarks` case arms executed in `ApiServer.Handle`, no embedded-asset load. Delta vs the pre-patch baseline: 2 bytes for the settings bools plus one `if (webViewsOn)` branch in `RadarApp.Run()`.
- **Zero-cost-when-idle contract.** With a toggle on but no browser connected: `SseChannel.Publish(state)` short-circuits on `_subscribers.IsEmpty`. Heartbeat timer is created lazily on the first subscriber add and disposed on last removal — no always-on timer when idle.
- **No new dedicated threads.** `SseChannel.Publish` runs on the world thread. Per-request handling and heartbeats use ThreadPool workers via `Task.Run` and `Timer`. If typical subscriber count grows past ~4 concurrent, `ThreadPool.SetMinThreads` is revisited in a follow-up; not needed for v0.20.0.
- **Version →** `0.20.0` in [`POE2Radar.Overlay.csproj`](../../../src/POE2Radar.Overlay/POE2Radar.Overlay.csproj).
- **CI gates unchanged:** `compliance-gate.ps1` + `scrub-strings.ps1 -SelfTest` must pass; full test suite green.
- **Pre-release testing.** Cut GitHub prerelease `v0.20.0-rc.1` (unlisted), hand-deliver via Discord DM to the two requesting testers (map upgrade + CN recon), iterate, then promote to public. Matches current RC practice — no in-app RC-channel infra changes.

### 2. `SseChannel` — subscriber fan-out with backpressure

`src/POE2Radar.Overlay/Web/SseChannel.cs` **(NEW, ~180 LOC)**. Owned by `RadarApp`; injected into `ApiServer`.

```csharp
sealed class Subscriber {
    public Guid Id;
    public HttpListenerResponse Response;
    public SemaphoreSlim WriteLock = new(1, 1);
    public int Queued;                 // outstanding writes; capped at MaxQueued
    public int ConsecutiveDrops;       // → force-disconnect at threshold
}

const int MaxQueued = 3;
const int MaxConsecutiveDrops = 30;    // 1 s of drops @ 30 Hz → boot slow client
const int HeartbeatSeconds = 15;

readonly ConcurrentDictionary<Guid, Subscriber> _subs = new();
Timer? _heartbeat;                     // lazy — created on first Add, disposed on last Remove
byte[]? _latest;                       // last serialized snapshot, for seed-on-connect
readonly object _latestLock = new();
```

- `HandleSubscribe(HttpListenerContext ctx)` sets `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Response.SendChunked = true`, `Response.KeepAlive = true`, then writes `:\n\n` to open. Adds to `_subs`. If `_latest != null` and non-empty, immediately writes it as the seed snapshot so the first paint is prompt (worst-case ~33 ms first-paint delay otherwise). Ensures the heartbeat timer is running (`Interlocked.CompareExchange` on `_heartbeat`). Awaits a `TaskCompletionSource` that completes when the client disconnects (write failure or `RequestAborted`).
- `Publish(RadarState state)` — early return if `_subs.IsEmpty`. Otherwise serializes once (`JsonSerializer.SerializeToUtf8Bytes(SnapshotForBrowser(state))`), formats as `data: <json>\n\n`, stores under `_latestLock` as `_latest`, then per-subscriber:
  - If `sub.Queued >= MaxQueued`, increment `ConsecutiveDrops`; if it exceeds `MaxConsecutiveDrops`, force-close the response and remove the subscriber. Do not enqueue this tick's write.
  - Otherwise `Interlocked.Increment(ref sub.Queued)`, `Task.Run(async () => { await sub.WriteLock.WaitAsync(); try { await sub.Response.OutputStream.WriteAsync(bytes); await sub.Response.OutputStream.FlushAsync(); sub.ConsecutiveDrops = 0; } catch { RemoveSubscriber(sub); } finally { sub.WriteLock.Release(); Interlocked.Decrement(ref sub.Queued); } })`.
- `RemoveSubscriber(Subscriber)` — remove from dict; if `_subs.IsEmpty` after removal, dispose `_heartbeat` and null it.
- Heartbeat callback writes `: ping\n\n` (SSE comment, client-ignored) to every subscriber to defeat proxy idle timeouts. The heartbeat cadence is orthogonal to the world tick.
- `SnapshotForBrowser(RadarState state)` projects `_state` into the browser wire format described in §4. Rewards list on monoliths is dropped for payload size — `bestName` + `bestEx` are retained for label rendering.

**Subscriber connect ordering:** a subscriber added to `_subs` between two `Publish` calls receives (a) the seed snapshot immediately on connect if one exists, and (b) the next `Publish` tick's snapshot. The seed guarantees a paint within milliseconds; there is no gap where the connected client sees nothing until the world thread ticks.

**Heartbeat test injection:** heartbeat cadence is driven by a `TimeProvider` field defaulted to `TimeProvider.System`. The test project injects `FakeTimeProvider` from `Microsoft.Extensions.TimeProvider.Testing` (test-only NuGet, not shipped).

### 3. `ApiServer` — gated case arms, multithread, gzip

[`src/POE2Radar.Overlay/Web/ApiServer.cs`](../../../src/POE2Radar.Overlay/Web/ApiServer.cs).

Today's dispatcher is a monolithic `switch (path)` inside `Handle(HttpListenerContext ctx)` starting at [line 200](../../../src/POE2Radar.Overlay/Web/ApiServer.cs#L200). We do **not** introduce a new dispatch table. We add gated case arms:

```csharp
// Inside Handle(ctx), same switch(path):
case "/map":
    if (!_settings.EnableWebMap) { NotFound(ctx); break; }
    _assetHost!.ServeMap(ctx);
    break;
case "/obs":
    if (!_settings.EnableWebObs) { NotFound(ctx); break; }
    _assetHost!.ServeObs(ctx);
    break;
case "/stream":
    if (!_settings.EnableWebMap && !_settings.EnableWebObs) { NotFound(ctx); break; }
    await _sse!.HandleSubscribe(ctx);
    break;
case "/api/map":
    if (!_settings.EnableWebMap && !_settings.EnableWebObs) { NotFound(ctx); break; }
    HandleTerrain(ctx);
    break;
case "/api/atlas":  // (existing arm at line 666 — add the gate at its top)
case "/landmarks":  // (existing arm at line 298 — add the gate at its top)
    // ...existing bodies, prefixed with a `webViewsOn` check as above.
    break;
```

Rationale: gating `/api/map`, `/api/atlas`, `/landmarks` behind **either** toggle (not just `EnableWebMap`) is required — otherwise a user who enables only `/obs` sees a broken render because the renderer's data endpoints don't exist. When both toggles are off, all five cases hit `NotFound(ctx)` before touching `_assetHost` or `_sse` (both null in that state).

`/assets/{name}` handling: the existing switch has no prefix routing. Add a new arm early in the switch:

```csharp
default:
    if ((_settings.EnableWebMap || _settings.EnableWebObs)
        && path.StartsWith("/assets/", StringComparison.Ordinal)) {
        _assetHost!.ServeAsset(ctx, path.Substring("/assets/".Length));
        break;
    }
    NotFound(ctx);
    break;
```

**Multithread request handling.** `Loop()` migrates from `_listener.GetContext()` + serial handling to `while (_running) { var ctx = await _listener.GetContextAsync(); _ = Task.Run(() => HandleContext(ctx)); }`. Individual handlers stay largely unchanged; they no longer share a call stack, so a slow `/api/map` payload cannot block `/state` or `/entities`. Existing single-threaded assumptions in the handlers are audited during implementation — none of the shared state (`_state` snapshot, settings, atlas cache) is mutated by GET handlers, so the migration is mechanically safe.

**Gzip pipeline.** When the incoming request carries `Accept-Encoding: gzip`, wrap the outgoing response stream in `GZipStream(CompressionLevel.Fastest)` and set `Content-Encoding: gzip`. Applied to `/api/map`, `/api/atlas`, `/landmarks`. **Not applied to `/stream`** — chunked SSE + gzip introduces flush-boundary complexity we don't need; raw `data: <json>\n\n` messages are small enough on their own (typical mid-map < 6 KB per message, comfortably under any per-message ceiling).

### 4. Wire format on `/stream`

One SSE message per world `Tick()` when at least one subscriber exists. Body:

```json
{
  "t": 1783391523412,
  "area": "abc123",
  "player": {"x": 512.4, "y": 384.7, "hp": 4210, "hpMax": 5000, "es": 300, "esMax": 400, "mana": 220, "manaMax": 300},
  "entities": [
    {"id": 42817, "x": 610.1, "y": 355.9, "cat": "hostile", "rar": "rare", "hp": 320, "hpMax": 400}
  ],
  "monoliths": [
    {"x": 480.0, "y": 400.0, "holes": 3, "unique": false, "collected": false, "anchor": "ChaosOrb", "bestEx": 42.0, "bestName": "Divine Orb", "color": "#33ff66"}
  ]
}
```

- **`t`** — server clock as `long`, milliseconds since Unix epoch (`DateTimeOffset.UtcNow.ToUnixTimeMilliseconds()`). Never in seconds. All client math treats `t` as ms to match `performance.now()` semantics.
- **`area`** — `areaHash` from `_state`. On change the client refetches terrain + atlas + landmarks and clears fog + position history.
- **`player`** — projected from `_state.Player`; fields flattened.
- **`entities`** — filtered from `_state.Entities`, capped at **800 items per snapshot** (raised from `/entities`'s 600). **Selection rule:** nearest-to-player by Euclidean distance in grid units; ties broken by ascending `Id` for stable ordering across ticks. **Dead-skip predicate:** `entity.hp <= 0 AND entity.hpMax > 0` — sentinel entities like POIs and containers (`hp == 0 && hpMax == 0`) are kept.
  - Enum values are locked here:
    - `cat ∈ { "hostile", "friendly", "npc", "poi" }`. `friendly` and `npc` render identically in v0.20.0 (see §7 layer 8) but stay distinct in the wire format for future differentiation.
    - `rar ∈ { "normal", "magic", "rare", "unique" }`. Meaningful only when `cat == "hostile"`; ignored otherwise.
- **`monoliths`** — projection of `_state.Monoliths` (`MonolithMarker` per [`ApiServer.cs:247-252`](../../../src/POE2Radar.Overlay/Web/ApiServer.cs#L247-L252)). `rewards[]` list is dropped for payload size; `bestName` truncated to 40 chars for label rendering.

Delta compression (diff entities by `Id`, omit unchanged `x`/`y`) is a v2 optimization; v1 sends full at 30 Hz. Typical mid-map raw payload < 6 KB per message; over a whole second that's ~180 KB/s of local-loopback traffic — trivial.

### 5. `AssetHost` — embedded resources

| `Web/Assets/atlas-icons.json` | The 15 atlas icon PNGs, base64-encoded inline. `MapPageHtml.cs` is **deleted** in this change. All references (constructed at [`ApiServer.cs:216-218`](../../../src/POE2Radar.Overlay/Web/ApiServer.cs#L216-L218)) are replaced by the new gated case arms in §3.

### 6. Client rendering (`map.js`)

Vanilla ES2020, no framework. Structure:

```
init()                          // parse ?gps=1 for /obs; load persisted zoom/isoMode/gpsMode; open EventSource; start rAF
onSseMessage(evt)               // parse; push to ring; on first message capture serverOffset; on area change → onZoneChange
onZoneChange(areaHash)          // clear fogCanvas to fully-unrevealed; flush position history; fetch /api/map, /api/atlas, /landmarks
frame(now)                      // rAF: renderTime = now + serverOffset - RENDER_DELAY_MS; find bracketing samples; lerp; draw
draw(pose, snapshot)            // z-ordered layer draw (see §7)
computeFogReveal(playerCell)    // marks reveal radius into fogCanvas; no-op in GPS mode
```

**Constants:**
```js
const RENDER_DELAY_MS = 66.67;      // ~2 * (1000/30) — two sample intervals of interpolation cushion
const REVEAL_RADIUS_CELLS = 24;     // matches in-game minimap reveal radius (~30 world units)
const RING_SIZE = 6;                // ~200 ms history at 30 Hz — enough for lerp (2), heading (3-5), and 1 jitter slot
const COS = 0.780430;               // matches C# MapProjection CameraCos
const SIN = 0.625243;               // matches C# MapProjection CameraSin
```

**Clock alignment (required, not implicit):**

```js
let serverOffset = 0;               // serverT - clientT (both ms)
const OFFSET_EMA_ALPHA = 0.2;       // gentle EMA to smooth network jitter
onSseMessage(evt) {
  const s = JSON.parse(evt.data);
  const clientT = performance.now();
  const rawOffset = s.t - clientT;
  serverOffset = (serverOffset === 0)
    ? rawOffset
    : serverOffset + OFFSET_EMA_ALPHA * (rawOffset - serverOffset);
  ring.push({ t: s.t, snap: s });
  // ...on area change → onZoneChange
}

// Each rAF frame:
const renderTime = performance.now() + serverOffset - RENDER_DELAY_MS;
const [a, b] = findBracketingSamples(ring, renderTime);
// If either missing → hold last pose (do NOT extrapolate).
```

**Ring is a single 6-slot buffer** shared by both the lerp-bracketing pass (uses the two samples straddling `renderTime`) and the heading estimator (§7.2, uses the last 3-5 for velocity). Two purposes, one array — no duplicated storage.

**Zone-change behavior:** when `snap.area !== currentArea`, `onZoneChange(snap.area)` clears `fogCanvas` to fully-unrevealed, flushes the ring (except the just-arrived sample), and refetches terrain/atlas/landmarks.

**`visibilitychange` handler:** on `hidden`, close `EventSource` and cancel the pending `requestAnimationFrame`. On `visible`, reopen `EventSource`, clear the ring, keep `fogCanvas` unless `areaHash` differs on the next message. Prevents runaway resource use on tabbed-away browsers and prevents lerp against stale samples on wake.

**GPS-mode toggle:**
- On `/map`: a HUD button top-right labeled "GPS" (icon + tooltip). Click toggles `body.gps-mode`. Keyboard shortcut `G` mirrors it. Persisted to `localStorage.gps_mode` as `"1"` / `"0"`.
- On `/obs`: HUD hidden (per §5 CSS). Toggle available only via URL — `/obs?gps=1` sets `body.gps-mode` on init.
- Existing `+` / `-` zoom (1..16, default 4) and iso/top-down toggle are preserved. Top-down is retained because it's genuinely useful for phone / OBS 16:9 layouts even though the in-game camera is fixed iso. Persisted to `localStorage.zoom` and `localStorage.iso_mode` respectively.

### 7. Renderer — layer order, palette, terrain edges, fog

**Draw order per frame (bottom-up):**

1. **Background** — `rgba(0, 0, 0, 0.55)` full-viewport. Skipped when `body.obs` (already handled by CSS via the transparent body background).
2. **Terrain interior** — `interiorCanvas` blit with `setTransform(COS, -SIN, -COS, -SIN, tx, ty)`, matching `MapProjection.GridDeltaToMapDelta` at [`MapProjection.cs:29-30`](../../../src/POE2Radar.Core/Pathfinding/MapProjection.cs#L29-L30) (`X = mapScale * (dx - dy) * COS`, `Y = -mapScale * (dx + dy) * SIN`). Interior fill `rgba(24, 22, 26, 0.85)`.
3. **Terrain edges** — `edgeCanvas` blit with the same transform. Edge stroke `rgba(255, 232, 180, 0.95)`. See §7.1.
4. **Fog mask** — draw `fogCanvas` over the terrain layers with default `globalCompositeOperation = 'source-over'`. `fogCanvas` is pre-painted so that **revealed cells are transparent** and **unrevealed cells are `rgba(0, 0, 0, 0.9)`**. Simple source-over composites the black over unrevealed cells and leaves revealed cells visible. Skipped when `body.gps-mode`.
5. *(Path polylines are v0.20.1 work — see Future Work §11. Not drawn in v0.20.0.)*
6. **Landmarks** — small diamond glyph + `12px Consolas, monospace` label with 1 px black shadow at each landmark position. Fetched once per zone via `/landmarks`.
7. **Atlas icons** — blit the 15 atlas PNGs at node positions from `/api/atlas`. Icons decoded once from bundled `atlas-icons.json` at init.
8. **Entities** — Canvas 2D circles + shapes. Radius `Math.max(2, 3 * zoom)`. Off-screen entities within `max(viewportWidth, viewportHeight) * 0.5` **screen pixels** of the viewport edge get a triangular arrow at the edge pointing toward them, colored by hostility. See §7.3 for the enum → color mapping.
9. **Monoliths** — `#c8a24a` ring, radius 8 px. Centered white text: the `holes` count. Below-ring label: `bestName` (truncated to 40 chars) in `#f0e6c8`, and `bestEx` formatted `Nex` where N is the numeric value, in dimmer `#c8a24a`. `collected == true` monoliths dim to `rgba(200, 162, 74, 0.4)` and skip the label.
10. **Player marker** — see §7.2.
11. **HUD text** — top-left area name · dot count · view mode · zoom. Skipped when `body.obs` (CSS-hidden).

#### 7.1. Terrain edge port

[`TerrainBitmap.BuildFrom`](../../../src/POE2Radar.Overlay/Overlay/TerrainBitmap.cs#L55-L101) is **8-neighbor Moore neighborhood** edge detection (confirmed at [lines 72-84](../../../src/POE2Radar.Overlay/Overlay/TerrainBitmap.cs#L72-L84): the `for (dy = -1..1) for (dx = -1..1)` double loop skips the center cell and checks all 8 neighbors, with out-of-bounds counted as wall). Port to JS as `buildEdges(walkable, width, height) → Uint8Array`. A cell is marked "edge" when `walkable[i] == 1` and **any of its 8 Moore-neighborhood cells (4 cardinal + 4 diagonal) is `0`, with out-of-bounds cells counted as `0` (wall)**. Result is painted once per zone into `edgeCanvas`; per-frame draw is a single `drawImage`, not a per-frame edge recompute.

**Parity contract:** the JS port must produce a byte-identical alpha map to the C# reference on a shared fixture (§Testing).

#### 7.2. Player marker & facing (v1 velocity-derived)

Client keeps the same 6-slot ring from §6. Each rAF frame:

- Let `p0 = ring[n-3].snap.player`, `p2 = ring[n-1].snap.player` (the newest and third-newest samples, ~66 ms apart).
- **Grid units are game world coords**, matching what `_state.Player.X/Y` publishes.
- Compute `dx = p2.x - p0.x`, `dy = p2.y - p0.y`. If `sqrt(dx² + dy²) > 0.15` grid units (~a fifth of a walk step at typical speed), `heading = atan2(dy, dx)`; draw a 10 px isosceles triangle in `#ffffff` pointing along `heading`.
- Otherwise draw a 4 px filled circle in `#ffffff`. Matches how the in-game marker looks when the player stands still.
- Missing ring slots (fewer than 3 samples so far, or a slot cleared by zone change) → default to the circle.

**Worked example.** Player walks 10 grid units/sec. Between two samples 66 ms apart, displacement is `10 * 0.066 = 0.66` grid units — well above the `0.15` threshold. Facing engages within two ticks of movement start.

Real player-heading memory read is **deferred to a follow-up spec** (§11); it requires reverse-engineering the facing offset (not in [`Poe2Offsets.cs`](../../../src/POE2Radar.Core/Game/Poe2Offsets.cs)) and a companion offset-validation session.

#### 7.3. Entity color mapping

Draws are keyed by (`cat`, `rar`) per §4:

- `cat == "hostile"`:
  - `rar == "normal"` → fill `#ff4a4a`, no ring.
  - `rar == "magic"`  → fill `#6a8bff`, no ring.
  - `rar == "rare"`   → fill `#ffd52e`, 1 px `#ffd52e` outer ring.
  - `rar == "unique"` → fill `#ff7a1a`, 1 px `#ff7a1a` outer ring.
- `cat == "friendly"` or `cat == "npc"` → fill `#65b6ff`, no ring. (Collapsed to one visual bucket in v0.20.0; wire format preserves the distinction for future differentiation.)
- `cat == "poi"` → fill `#ffd85c`, plus a 6 px crosshair overlay (two 1 px lines at right angles).
- Dead-with-`hpMax` (predicate `hp <= 0 && hpMax > 0`) → not drawn. Skip happens on the client for defense-in-depth even though the server already filters (§4).

## Tencent CN Client — Recon Appendix

This appendix ships as a section of this design doc. **No code lands in v0.20.0 for CN compatibility.** The recon exists so a decision to build (or not build) the kernel-driver path can be made with real numbers, and so the requesting tester gets a clear "here's why it's blocked, here's what's needed to unblock" answer.

### The one-line reality

**TenSafe / ACE (Tencent's kernel-mode anti-cheat) blocks `OpenProcess(PROCESS_VM_READ)` at the object-manager layer.** Every downstream `NtReadVirtualMemory`, `EnumProcessModulesEx`, and `AobScanner.ReadExecutableSections` call fails from the same root cause. No userland workaround exists. This kills the app before any other CN-specific work matters.

### Full blocker inventory (ordered by dependency)

1. **AC layer** — [`ProcessHandle.cs:66-72`](../../../src/POE2Radar.Core/ProcessHandle.cs#L66-L72) `OpenProcess(PROCESS_VM_READ | PROCESS_QUERY_LIMITED_INFORMATION)` fails under TenSafe handle-stripping.
2. **Binary discovery** — [`ProcessHandle.cs:40, 95`](../../../src/POE2Radar.Core/ProcessHandle.cs#L40) hardcodes 5 candidate names, none Tencent/WeGame. CN PoE2 binary name unconfirmed as of 2026-07-06; PoE1 CN used `PathOfExile_x64Steam.exe` under WeGame but PoE2 CN has not stably launched.
3. **AOB brittleness under AC hooks** — [`AobPatterns.cs:36-46, 58-65, 88-96`](../../../src/POE2Radar.Core/Game/AobPatterns.cs#L36-L96). `FileRoot` relies on trailing `E8..E8` uniqueness; TenSafe injects `E8/E9` trampolines into hot code and breaks this even ignoring compiler drift.
4. **`.text` page-protection gaps** — [`AobScanner.cs:47-71`](../../../src/POE2Radar.Core/Game/AobScanner.cs#L47-L71) skips `PAGE_GUARD` regions; TenSafe stealth-page-swaps hot ranges, so the AOB scan runs on a truncated haystack and pointer-slot resolution returns zero hits.
5. **Offset drift risk** — every offset in [`Poe2Offsets.cs`](../../../src/POE2Radar.Core/Game/Poe2Offsets.cs) validated only against international. CN builds lag international patches; every offset needs a parallel CN table.
6. **CN-specific struct fields** — Tencent-SDK / MTX / AC structs may shift offsets even at matched patch versions. Unverified without a live sample.
7. **Locale-string logic bombs** — [`Poe2Live.cs`](../../../src/POE2Radar.Core/Game/Poe2Live.cs) HC/SC detection `league.StartsWith("HC ")` misclassifies every CN league as softcore (grep `Poe2Live.cs` for the `StartsWith("HC "` call site — the docstring at line 276-277 references it, and the actual check lives adjacent). [`Poe2Offsets.cs:513-516`](../../../src/POE2Radar.Core/Game/Poe2Offsets.cs#L513-L516) `WorldAreas +0x08` returns Simplified Chinese display name; English equality checks silently mismatch.
8. **Content tables** — `atlas_content.json`, `atlas_maps.json`, `world_areas.json` ship no `simplified chinese` locale. Would need harvesting from RePoE or CN Content.ggpk.
9. **Distribution** — GitHub Releases / `api.github.com` unreachable-or-rate-limited from mainland China ([`UpdateChecker.cs`](../../../src/POE2Radar.Overlay/Update/UpdateChecker.cs), [`AutoUpdatePolicy.cs`](../../../src/POE2Radar.Overlay/Update/AutoUpdatePolicy.cs)). Auto-update silently times out exactly when a CN patch invalidates offsets and a hotfix is most needed.
10. **Community pack endpoint** — `*.workers.dev` is blocked in China ([`RadarSettings.cs:75`](../../../src/POE2Radar.Overlay/Config/RadarSettings.cs#L75)). Contribute button dies for CN users (non-critical).

**Not blockers:** metadata paths are engine-internal ASCII (`Metadata/Monsters/...`), `AreaCode` is ASCII (`G1_town`), window discovery is title-agnostic (PID-based), no `Client.txt` parsing, no packet protocol, no auth, no GGPK/bundle assumptions. Ingest logic itself is locale-independent once the process is open and offsets are right.

### The kernel-mode reader path — high-level architecture

A minifilter or KMDF driver exposing `IOCTL_READ_PROCESS_MEMORY` to a userland service. POE2GPS talks to the service instead of calling `OpenProcess` directly. Driver runs at ring 0; TenSafe's userland handle-stripping doesn't reach it.

**Signing story:**
- Extended-Validation code-signing certificate (~$300-600/yr from Sectigo / DigiCert; requires business identity verification).
- Attestation-signed driver via Microsoft Partner Center (free once EV cert is registered; ~1-2 day turnaround per submission).
- Optional WHQL signing for wider compatibility (adds queue time; not strictly required for consumer install).

**Install path:** MSI or elevated installer with driver-load step. Users see a UAC prompt and a "Windows verified this driver" dialog. Driver is signed by Microsoft after attestation submission.

**Cost ballpark:**
- **Calendar:** 3-6 months of solo work (driver + service + IPC + userland refactor to talk to service).
- **Money:** ~$1500 first-year (EV cert + Partner Center registration + notary if legal review is wanted).
- **Maintenance:** driver code is now attack surface; needs updates as Windows kernel evolves; each POE2 patch that changes AC behavior requires driver testing.

### Vetted AC bypass — mentioned and rejected

Not because it's technically impossible, but because:
- It turns POE2GPS into something GGG-and-Tencent-both actively hunt. Any user detected running it in-game risks a ban.
- Distributing a bypass has real legal exposure (DMCA §1201 anti-circumvention; Chinese Cybersecurity Law Art. 27).
- It erodes the strictly-read-only trust posture that makes POE2GPS acceptable to GGG today.

Documented for completeness only. Not a path this project takes.

### Userland CN-prep work list (deferred)

Everything that could ship without a driver, ordered ready-to-execute the day a bypass or driver exists:

- Add candidate CN binary name to [`ProcessHandle.cs`](../../../src/POE2Radar.Core/ProcessHandle.cs) once a live sample confirms it.
- Dual-offset table in [`Poe2Offsets.cs`](../../../src/POE2Radar.Core/Game/Poe2Offsets.cs) keyed on client detection.
- Non-string HC/SC detection (identify a boolean flag in `ServerData` if one exists, avoiding the English `"HC "` prefix check).
- Simplified Chinese locale harvest into `atlas_content.json`, `atlas_maps.json`, `world_areas.json`.
- Configurable `UpdateUrl` in [`AutoUpdatePolicy.cs`](../../../src/POE2Radar.Overlay/Update/AutoUpdatePolicy.cs) — deferred to v0.20.1 (see §11).

**Honest assessment:** all of this ships zero user value until the AC blocker is solved. So don't do it now. Kept on the shelf, in order, ready to execute.

### Recommendation

Do not commit code time to CN until:
- CN PoE2 client publicly launches, **and**
- Its binary name, anti-cheat variant, and content-table shape are confirmed against a live sample.

Until then, this doc is the reference. The requesting tester gets a link to it plus an offer to test the /map upgrade if they're running the international client via VPN.
